<objective>
Fix peer discovery deduplication bug where mDNS scanning shows duplicate devices.

Purpose: Two root causes produce duplicates — (1) `get_discovered_peers()` does not filter `local_peer_id`, allowing self-discovery in edge cases; (2) daemon `peers.changed` emits single-peer increments but frontend replaces the entire peer list with each event, causing flash/duplicate artifacts. Fix both at backend layer per D-04 decision.

Output: Patched `get_discovered_peers()` with local_peer_id exclusion, daemon `peers.changed` emits full snapshot, DaemonWsBridge translates full payload, regression tests for both fixes.
</objective>
