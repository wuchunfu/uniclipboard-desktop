## 0.3.0-alpha.5 - 2026-03-15

### Features

- 新增基于 libp2p 的跨平台文件同步，支持分块传输、持久化跟踪与启动时状态对账。
- 新增持久化文件传输状态展示，在列表与预览中显示 pending/transferring/completed/failed 及失败原因。
- 新增文件条目的复制门禁逻辑，传输完成前禁用 Copy，同时保留 Delete 可用。
- 新增文件传输仓储与数据表升级，支持元数据优先跟踪、按条目聚合状态与超时恢复。
- 新增文件同步设置能力，覆盖开关、大小限制、保留周期、配额检查与启动清理任务。
- 新增入站与出站文件同步保护，处理同步关闭与超大文件场景。
- 新增快捷面板能力，用于快速访问剪贴板历史并强化键盘优先交互。
- 新增链接内容类型支持，覆盖 MIME link 与纯文本 URL 识别，并支持多 URL 展示。
- 新增 Seq 日志接入，将前端控制台日志汇聚并标记后端日志来源。
- 新增 macOS 解锁流程中的 Keychain Always Allow 确认步骤。

### Fixes

- 修复跨设备文件同步时重复条目问题，完善去重与竞态处理逻辑。
- 修复文件路径处理问题：统一将剪贴板文件路径规范为 `file://` URI，并在写入时转换回原生路径。
- 修复传输记录卡在 pending 的问题，补齐提前完成对账并可靠持久化 pending 记录。
- 修复传输状态 Tooltip 渲染崩溃，补充 `TooltipProvider` 包装。
- 修复文件恢复后重复采集问题，改用 `LocalRestore` 剪贴板来源标记。
- 修复文件复制命令参数命名问题，统一使用 camelCase。
- 修复剪贴板文件写入问题，写入前将相对路径解析为绝对路径。
- 修复应用路径漂移问题，使应用目录命名与桌面端标识保持一致。

### Refactor

- 重构路径构建流程，统一通过 `AppPaths::from_app_dirs()` 作为单一来源。
- 重构快捷面板模块位置，将其迁移到 `uc-tauri` 以对齐架构边界。
- 重构链接检测路径，将逻辑前移到用例数据层，替代仅预览阶段解析。

### Tests

- 新增文件同步流程的 UAT 覆盖，全部场景通过。
- 更新并扩展传输状态跟踪、API hydration 与序列化行为测试覆盖。
