<objective>
把 46.3 的启动结果变成明确的 shell contract：主窗口展示必须受 daemon-compatible readiness 约束，失败必须进入 typed startup state，并对前端暴露可轮询、可手动重试的命令接口。

Purpose: 把“daemon bootstrap 只记日志、窗口仍提前展示”的现状替换成可验证的 startup gate 和恢复入口。
Output: 新的 startup state 模块、带 tracing 的 startup status/retry commands、main.rs 窗口 gating 逻辑、以及对应 Rust regression tests。
</objective>

  <action>
    1. 新增 `src-tauri/crates/uc-tauri/src/bootstrap/startup_state.rs`，定义固定状态模型：
       - `StartupPhase::{Bootstrapping, Ready, Failed}`
       - `StartupFailureCode::{IncompatibleDaemon, ReplaceFailed, SpawnFailed, ProbeTimeout, ConnectionInfoUnavailable}`
       - `StartupStatusSnapshot` 字段固定为 `phase`, `failure_code`, `message`, `attempt`, `can_retry`, `is_retrying`
       并给所有对前端可见的 DTO 加 `#[serde(rename_all = "camelCase")]`。
    2. 在 `src-tauri/crates/uc-tauri/src/bootstrap/mod.rs` 中导出 startup state 模块，供 commands/main 使用。
    3. 扩展 `src-tauri/crates/uc-tauri/src/commands/startup.rs` 里的 `StartupBarrier`：
       - 新增 `daemon_ready` 与 `startup_failed` 标志。
       - `try_finish()` 只有在 `backend_ready && daemon_ready && !startup_failed` 时才允许正常展示主窗口。
       - 保留 `frontend_ready` / `daemon_connection_emitted` 现有语义。
    4. 更新 `src-tauri/src/main.rs`：
       - 用 `.manage()` 注册新的 `StartupState`。
       - 移除 `.setup()` 中基于 `silent_start=false` 的直接 `show_main_window(app.handle())` 正常路径展示。
       - daemon bootstrap 成功时：`mark_daemon_ready()` 并让 `try_finish()` 决定何时展示主窗口。
       - daemon bootstrap 失败时：写入 `StartupPhase::Failed`，记录 `failureCode/message`，并且即使 `silent_start=true` 也强制展示主窗口，让错误页有机会渲染。
  </action>
