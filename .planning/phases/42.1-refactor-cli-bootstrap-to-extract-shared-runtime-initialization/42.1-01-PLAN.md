# Plan 42.1-01: Add shared CLI runtime helper to uc-bootstrap

## Goal

Create one reusable `uc-bootstrap` helper that owns the repeated direct-mode CLI runtime assembly path currently duplicated across command handlers. Preserve `uc-bootstrap` as the composition root and keep the phase behavior-preserving.

## Tasks

<task id="1">
<name>Add CLI runtime helper APIs to uc-bootstrap</name>
<read_first>
- src-tauri/crates/uc-bootstrap/src/lib.rs
- src-tauri/crates/uc-bootstrap/src/non_gui_runtime.rs
- src-tauri/crates/uc-bootstrap/src/builders.rs
- .planning/phases/42.1-refactor-cli-bootstrap-to-extract-shared-runtime-initialization/42.1-RESEARCH.md
</read_first>
<action>
Update `src-tauri/crates/uc-bootstrap/src/non_gui_runtime.rs` to add these public APIs:

```rust
pub fn build_cli_runtime() -> anyhow::Result<uc_app::runtime::CoreRuntime>
pub fn build_cli_runtime_with_profile(
    log_profile: Option<uc_observability::LogProfile>,
) -> anyhow::Result<uc_app::runtime::CoreRuntime>
```

Inside `build_cli_runtime_with_profile(log_profile)` implement the current bootstrap sequence in this exact order:

1. `uc_bootstrap::build_cli_context_with_profile(...)`
2. `uc_bootstrap::get_storage_paths(&ctx.config)`
3. `uc_bootstrap::build_non_gui_runtime(ctx.deps, storage_paths)`

Because the helper is being added inside `uc-bootstrap`, use crate-local calls rather than going through the external re-export path:

1. `crate::build_cli_context_with_profile(log_profile)`
2. `crate::get_storage_paths(&ctx.config)`
3. `build_non_gui_runtime(ctx.deps, storage_paths)`

Wrap each failure with `anyhow::Context` using these exact messages:

- `failed to initialize CLI context`
- `failed to resolve storage paths`
- `failed to build CLI runtime`

Implement `build_cli_runtime()` as a thin default wrapper that calls:

```rust
build_cli_runtime_with_profile(Some(uc_observability::LogProfile::Cli))
```

Do not construct `CoreUseCases` inside the helper. Return only `CoreRuntime`; command handlers will continue to call `CoreUseCases::new(&runtime)` locally when needed.

Update `src-tauri/crates/uc-bootstrap/src/lib.rs` to re-export both new functions.
</action>
