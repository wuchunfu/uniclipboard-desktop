**Code to insert (per D-01, D-02, D-05, D-06, D-08):**

```rust
// 1.5 Recover encryption session (fail-fast per D-05: must succeed if Initialized)
{
    let span = tracing::info_span!("daemon.startup.recover_encryption_session");
    async {
        let uc = self.runtime.usecases().auto_unlock_encryption_session();
        match uc.execute().await {
            Ok(true) => {
                tracing::info!("Encryption session recovered from disk");
            }
            Ok(false) => {
                tracing::info!("Encryption not initialized, skipping session recovery");
            }
            Err(e) => {
                tracing::error!(error = %e, "Encryption session recovery failed");
                anyhow::bail!(
                    "Cannot start daemon: encryption session recovery failed: {}",
                    e
                );
            }
        }
        Ok::<(), anyhow::Error>(())
    }
    .instrument(span)
    .await?;
}
```

**Also add to imports at top of file:**

- Add `Instrument` to the existing `tracing` import: change `use tracing::{info, warn};` to `use tracing::{info, info_span, warn, Instrument};`

**Also add to the existing `use tracing::` line, add `error`** so it becomes: `use tracing::{error, info, info_span, warn, Instrument};`

**Key design points:**

- Uses `match` (not `if let`) per CLAUDE.md error handling rules (D-05 requires reporting failure)
- `Ok(false)` is not an error (D-08: Uninitialized means first run, skip recovery)
- `Err(e)` calls `anyhow::bail!` to refuse startup (D-05, D-06: no degraded mode)
- `info_span!("daemon.startup.recover_encryption_session")` follows project span naming convention
- The block is scoped in `{}` so the use case temporary is dropped before workers start
