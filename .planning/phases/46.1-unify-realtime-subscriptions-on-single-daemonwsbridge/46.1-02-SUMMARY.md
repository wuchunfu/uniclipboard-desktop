# Phase 46.1 Plan 02: App Realtime Consumers And Setup Event Hub Summary

**`uc-app` now owns pairing, peers, and setup realtime consumption through typed ports instead of feature-owned websocket clients**

## Performance

- **Duration:** 5 min
- **Started:** 2026-03-20T21:53:42+0800
- **Completed:** 2026-03-20T21:58:31+0800
- **Tasks:** 2
- **Files modified:** 7

## Accomplishments

- Added dedicated `uc-app::realtime` consumers for pairing, peers, and setup, each subscribing through `RealtimeTopicPort`.
- Mapped pairing and peers realtime updates into `HostEvent::Realtime(RealtimeFrontendEvent)` so downstream Tauri adapters can emit the single `daemon://realtime` contract without daemon DTO leakage into `uc-app`.
- Added `SetupPairingEventHub` to fan out `PairingDomainEvent` receivers from one shared realtime feed, including per-session ordering guards that drop regressive events after terminal updates.
- Landed focused uc-app tests covering pairing envelope mapping, peer delta emission, setup fanout reuse, and session-order filtering.

## Decisions Made

- Kept `PairingDomainEvent` as the setup-facing contract so the setup flow can migrate to shared realtime delivery without rewriting downstream orchestrator logic.
- Treated per-session ordering as an app concern, not a bridge concern, so later bridge work can stay transport-focused while `uc-app` guards business-visible regressions.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Aligned failed-pairing mapping with the current core realtime contract**

- **Found during:** Task 2 (Implement uc-app realtime consumers and a shared setup pairing event hub)
- **Issue:** `PairingFailedEvent` in `uc-core::ports::realtime` currently carries only `session_id` and `reason`, so the initial setup consumer implementation incorrectly expected a `peer_id` field and failed to compile.
- **Fix:** Mapped failed setup events with an empty `peer_id` placeholder while preserving the existing `PairingDomainEvent::PairingFailed` contract and `FailureReason::Other(reason)` semantics.
- **Files modified:** `src-tauri/crates/uc-app/src/realtime/setup_consumer.rs`
- **Verification:** `cd src-tauri && cargo test -p uc-app realtime_consumer -- --test-threads=1`
