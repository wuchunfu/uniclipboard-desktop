The on_clipboard_changed() method currently has 3 inline stages:

## File Candidate Construction Contract

The runtime currently does:

1. `extract_file_paths_from_snapshot()` parses URIs AND resolves APFS -> `Vec<PathBuf>`
2. `std::fs::metadata()` on each path to get file size for max_file_size filtering

After this plan, the runtime will:

1. `extract_file_paths_from_snapshot()` STAYS in runtime.rs (unchanged, still resolves APFS) -> `Vec<PathBuf>`
2. **Record `extracted_paths_count = resolved_paths.len()` BEFORE metadata filtering**
3. For each resolved path, call `std::fs::metadata()` to get file size, build `FileCandidate { path, size }`
4. Files where `metadata()` fails are excluded at this stage (with tracing::warn), matching current behavior
5. Pass `Vec<FileCandidate>` AND `extracted_paths_count` to `planner.plan()` -- planner uses both to detect all_files_excluded
6. Runtime dispatches based on plan.clipboard / plan.files

This keeps ALL filesystem I/O in the platform layer (uc-tauri) where it belongs.
The planner in uc-app remains a pure function with zero filesystem dependencies.

## LocalRestore Semantics

Current code (sync_outbound.rs:177): only RemotePush is skipped, LocalRestore triggers clipboard sync.
The planner preserves this: plan() returns clipboard: Some for LocalRestore.
For LocalRestore, runtime passes empty file_candidates and extracted_paths_count=0 (no file sync for non-LocalCapture).

   Extract file paths BEFORE calling the planner, gated on origin == LocalCapture:

```rust
let resolved_paths = if origin == ClipboardChangeOrigin::LocalCapture {
    extract_file_paths_from_snapshot(&outbound_snapshot)

```

2. **Record extracted_paths_count BEFORE metadata filtering.** Then build `Vec<FileCandidate>`
   from resolved paths by calling `std::fs::metadata()` on each. This keeps ALL filesystem I/O
   in the platform layer (uc-tauri), not in the planner (uc-app). Files where metadata() fails
   are excluded with a tracing::warn, matching current behavior:

```rust
use uc_app::usecases::sync_planner::FileCandidate;

let extracted_paths_count = resolved_paths.len();

let file_candidates: Vec<FileCandidate> = resolved_paths
    .into_iter()
    .filter_map(|path| {
        match std::fs::metadata(&path) {
            Ok(meta) => Some(FileCandidate {
                path,
                size: meta.len(),
            }),
            Err(e) => {
                tracing::warn!(
                    error = %e,
                    file = %path.display(),
                    "Excluding file from sync: failed to read metadata"
                );
                None
            }
        }
    })
    .collect();
```

3. Create the planner and call plan() with the pre-computed file candidates AND extracted_paths_count:

```rust
let planner = uc_app::usecases::sync_planner::OutboundSyncPlanner::new(
    self.deps.settings.clone(),
);
let plan = planner.plan(outbound_snapshot, origin, file_candidates, extracted_paths_count).await;
```

4. Dispatch clipboard sync from plan.clipboard:

```rust
if let Some(clipboard_intent) = plan.clipboard {
    let outbound_sync_uc = self.usecases().sync_outbound_clipboard();
    let flow_id_for_sync = flow_id.clone();
    let flow_id_str = flow_id_for_sync.to_string();
    tauri::async_runtime::spawn(
        async move {
            match tokio::task::spawn_blocking(move || {
                outbound_sync_uc.execute(
                    clipboard_intent.snapshot,
                    origin,
                    Some(flow_id_str),
                    clipboard_intent.file_transfers,
                )
            })
            .await
            {
                Ok(Ok(())) => {
                    tracing::info!("Outbound clipboard sync completed");
                }
                Ok(Err(err)) => {
                    tracing::warn!(error = %err, "Outbound clipboard sync failed");
                }
                Err(err) => {
                    tracing::warn!(error = %err, "Outbound clipboard sync task join failed");
                }
            }
        }
        .instrument(tracing::info_span!("outbound_sync", %flow_id_for_sync)),
    );
}
```
