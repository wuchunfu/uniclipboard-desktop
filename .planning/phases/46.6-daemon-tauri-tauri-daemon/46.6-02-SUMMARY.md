**Tauri real-exit gating now boundedly tears down GUI-owned daemon children and locks the contract with exact integration regressions**

## Accomplishments

- Added a bounded `shutdown_owned_daemon()` path that terminates, waits, and force-kills GUI-owned daemon children only when graceful shutdown times out.
- Rewired `RunEvent::ExitRequested` to cancel tracked tasks, prevent real exit during cleanup, and reissue `app.exit(0)` only after cleanup settles.
- Replaced placeholder daemon exit cleanup coverage with real regression tests for spawned-owned cleanup, compatible no-op, replacement-owned cleanup, repeated exit idempotency, and tray-hide no-op semantics.

## Decisions Made

- Real daemon cleanup remains attached only to `RunEvent::ExitRequested`; `WindowEvent::CloseRequested` continues to be a tray-hide path with no daemon termination calls.
- `GuiOwnedDaemonState` now owns the cleanup lifecycle: it takes the child for teardown, restores ownership on failure for observability/retry, and only clears state after successful shutdown.
- Regression coverage verifies the contract through exact `cargo test --test ...` targets to avoid substring-filter false greens.
