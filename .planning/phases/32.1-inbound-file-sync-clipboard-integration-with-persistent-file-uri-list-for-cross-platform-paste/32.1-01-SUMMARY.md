## Decisions Made

- CopyFileToClipboardUseCase takes only entry_id and internally loads the entry to get event_id, keeping the Tauri command simple
- Batch accumulator state (HashMap) lives in the event loop outside tokio::spawn so it can coordinate across multiple FileTransferCompleted events
- Single-file transfers write clipboard immediately; batch transfers wait for all files
- Entry persistence always happens regardless of clipboard race (user can Copy from Dashboard even if auto-write was skipped)
- Added get_representations_for_event to ClipboardRepresentationRepositoryPort with a default empty-vec implementation to avoid breaking existing trait implementations
