## 0.3.0-alpha.5 - 2026-03-15

### Features

- Add cross-platform file sync over libp2p with chunked transfer, durable tracking, and startup reconciliation.
- Add durable file transfer states to list and preview, including pending/transferring/completed/failed and failure reasons.
- Add copy gating for file entries so Copy is blocked until transfer completion while Delete remains available.
- Add file transfer repository and schema upgrades for metadata-first tracking, entry-level aggregation, and timeout recovery.
- Add file sync settings for enablement, size limits, retention, quota checks, and startup cleanup tasks.
- Add inbound and outbound file sync guards for disabled sync and oversized files.
- Add quick panel support for clipboard history access with keyboard-first interactions.
- Add link content type support for MIME links and URL-detected plain text, including multi-URL rendering.
- Add Seq log ingestion for frontend console logs with backend source tagging.
- Add macOS Keychain Always Allow confirmation flow on unlock.

### Fixes

- Fix duplicate file entries during cross-device sync by improving deduplication and race handling.
- Fix file path handling by normalizing clipboard file paths to `file://` URIs and converting back to native paths for writes.
- Fix transfer records stuck in pending by reconciling early completions and persisting pending records reliably.
- Fix frontend crash in transfer status tooltips by wrapping tooltip usage with `TooltipProvider`.
- Fix duplicate file captures after restore by using `LocalRestore` clipboard origin handling.
- Fix file copy command payload naming by using camelCase parameters.
- Fix clipboard file writes by resolving relative paths to absolute paths before write.
- Fix app path drift by aligning app directory naming with the desktop identifier.

### Refactor

- Refactor path construction to use `AppPaths::from_app_dirs()` as a single source of truth.
- Refactor quick panel module placement into `uc-tauri` to align with architecture boundaries.
- Refactor link detection to the use-case data path instead of preview-only parsing.

### Tests

- Add UAT coverage for file sync flows with all scenarios passing.
- Update and expand test coverage for transfer tracking, API hydration, and serialization behavior.

### Docs

- Add and complete planning/summary documentation for phases 28-32 and the related gap-closure work.
- Document phase renumbering and execution tracking updates for file sync delivery.
