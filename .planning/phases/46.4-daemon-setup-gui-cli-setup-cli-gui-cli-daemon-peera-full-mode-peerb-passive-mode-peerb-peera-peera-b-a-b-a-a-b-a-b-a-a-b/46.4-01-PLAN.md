  <action>
    1. 在 `src-tauri/crates/uc-daemon/src/api/types.rs` 中新增 camelCase setup DTO，固定包含这些 transport 形态：
       - `SetupStateResponse { state, session_id, next_step_hint, profile, clipboard_mode, device_name, peer_id, selected_peer_id, selected_peer_name, has_completed }`
       - `SetupSelectPeerRequest { peer_id }`
       - `SetupSubmitPassphraseRequest { passphrase }`
       - `SetupActionAckResponse { state, session_id, next_step_hint }`
       所有新 DTO 都必须加 `#[serde(rename_all = "camelCase")]`。
    2. 在 `src-tauri/crates/uc-daemon/src/api/server.rs` 中把 setup orchestration access 注入 `DaemonApiState`，让 route handler 能直接拿到 daemon runtime 内部的 `SetupOrchestrator`，不要在 handler 里新建 runtime，也不要把 setup 逻辑塞回 CLI 或 HTTP 层。
    3. 在 `src-tauri/crates/uc-daemon/src/api/query.rs` 中新增 setup state query 组装逻辑，用现有 runtime/usecase 信息填充 `profile`、`clipboard_mode`、`device_name`、`peer_id`、`session_id`、`has_completed`、`next_step_hint`；`next_step_hint` 必须返回固定字符串集合中的一个：
       - `create-space-passphrase`
       - `join-select-peer`
       - `join-waiting-for-host`
       - `join-enter-passphrase`
       - `host-confirm-peer`
       - `completed`
       - `idle`
    4. 在 `src-tauri/crates/uc-daemon/src/api/routes.rs` 中注册并实现这些授权后的 setup 路由：
       - `GET /setup/state`
       - `POST /setup/host`
       - `POST /setup/join`
       - `POST /setup/select-peer`
       - `POST /setup/confirm-peer`
       - `POST /setup/submit-passphrase`
       - `POST /setup/cancel`
       这些 handler 必须分别委派到 `get_state()`、`new_space()`、`join_space()`、`select_device(peer_id)`、`confirm_peer_trust()`、`cancel_setup()`；其中 `POST /setup/submit-passphrase` 必须先读取当前 setup state，再按固定规则委派：
       - 当前 state 是 create-space passphrase 阶段 -> `submit_passphrase(passphrase.clone(), passphrase)`
       - 当前 state 是 join-space passphrase 阶段 -> `verify_passphrase(passphrase)`
       - 其他 state -> `409 invalid_setup_transition`
       不要在 route handler 里复制 state machine 判断。
    5. 错误映射固定为：
       - 未授权 -> `401`
       - request body 缺失或损坏 -> `400` + JSON `{"code":"bad_request","message":"..."}`
       - 当前 setup 状态不允许该操作 -> `409` + JSON `{"code":"invalid_setup_transition","message":"..."}`
       - 其他 usecase/internal failure -> `500` + JSON `{"code":"setup_failed","message":"..."}`
    6. 在 `src-tauri/crates/uc-daemon/src/app.rs` 中把 setup orchestration 依赖传入 API server/state 装配，让 `/setup/*` surface 真正绑定 daemon 进程内的单一 setup truth。
