Wire inbound file transfer completion to system clipboard write with batch accumulation, clipboard entry persistence, clipboard race detection, and create CopyFileToClipboardUseCase for manual Dashboard copies.

  <action>
    **1. Create CopyFileToClipboardUseCase** (`copy_file_to_clipboard.rs`):

    Architecture decision (per checker issue #6): CopyFileToClipboardUseCase takes `entry_id` and internally accesses `ClipboardEntryRepositoryPort` + `ClipboardRepresentationRepositoryPort` to load the entry's text/uri-list content. This avoids the Tauri command needing to know about representation loading.

    - Struct fields:
      - `entry_repo: Arc<dyn ClipboardEntryRepositoryPort>`
      - `representation_repo: Arc<dyn ClipboardRepresentationRepositoryPort>`
      - `local_clipboard: Arc<dyn SystemClipboardPort>`
      - `clipboard_change_origin: Arc<dyn ClipboardChangeOriginPort>`
      - `mode: ClipboardIntegrationMode`
    - `pub async fn execute(&self, entry_id: &EntryId) -> Result<()>`:
      1. Check `self.mode.allow_os_write()` -- return error if disabled
      2. Load representations for entry_id from `self.representation_repo`
      3. Find representation with mime_type `text/uri-list` or `file/uri-list` -- return error if not found
      4. Decode bytes to string, parse `file://` URIs using `url::Url::parse(line)?.to_file_path()`
      5. Validate each file path exists with `tokio::fs::metadata(&path).await` -- if ANY missing, return `Err(anyhow!("file deleted"))`
      6. Build `SystemClipboardSnapshot` with single representation: mime_type `text/uri-list`, format_id `files`, bytes = URI list string bytes
      7. Set origin: `self.clipboard_change_origin.set_next_origin(ClipboardChangeOrigin::RemotePush, Duration::from_secs(2)).await`
      8. Call `self.local_clipboard.write_snapshot(snapshot)` -- on error, consume origin back to LocalCapture (follow restore_clipboard_selection.rs pattern)
    - `pub async fn execute_from_paths(&self, file_paths: Vec<PathBuf>) -> Result<()>`:
      1. Validate each file path exists
      2. Build URI list and snapshot
      3. Set origin and write snapshot (same steps 6-8 as above)
    - Export from `mod.rs` with `pub use copy_file_to_clipboard::CopyFileToClipboardUseCase;`

    **3. Wire clipboard write with batch accumulation and entry persistence in wiring.rs**:

    Add to `run_pairing_event_loop` parameters:
    - `system_clipboard: Arc<dyn SystemClipboardPort>`
    - `clipboard_change_origin: Arc<dyn ClipboardChangeOriginPort>`
    - `capture_clipboard_uc: Arc<CaptureClipboardUseCase>` (for entry persistence)

    Create a batch accumulator state OUTSIDE the event loop (before `while let Some(event) = event_rx.recv().await`):
    ```rust
    // Batch accumulator: batch_id -> (completed_paths: Vec<PathBuf>, expected_total: Option<u32>)
    let mut batch_accumulator: HashMap<String, (Vec<PathBuf>, Option<u32>)> = HashMap::new();
    ```

    In the `FileTransferCompleted` handler, after `handle_transfer_complete()` succeeds:

    a. **Batch detection**: If `batch_id` is `Some(bid)` and `batch_total` is `Some(total)`:
       - Insert/append `result.file_path` to `batch_accumulator[bid]`
       - If accumulated count < total: log "Batch {bid}: {count}/{total} files received, waiting for remaining" and continue (do NOT write clipboard yet)
       - If accumulated count == total: collect all paths, remove batch from accumulator, proceed to clipboard write with all paths
    b. **Single file** (batch_id is None): proceed directly to clipboard write with single path

    c. **Clipboard race detection (FCLIP-03)**: Before writing to clipboard, check if the user copied something during the transfer. Use `consume_origin_or_default`:
       ```rust
       let current_origin = clipboard_change_origin
           .consume_origin_or_default(ClipboardChangeOrigin::LocalCapture)
           .await;
       if current_origin == ClipboardChangeOrigin::LocalCapture {
           // User has NOT copied anything -- safe to auto-write
       } else {
           // Origin was already set (e.g., by a recent user copy that set LocalCapture),
           // meaning user is actively using clipboard -- cancel auto-write
           info!("Clipboard race detected: user copied during transfer, skipping auto-write");
           // Still persist the entry (see step d below) so user can Copy from Dashboard
       }
       ```
       NOTE: The above is a heuristic. A more robust approach: set `ClipboardChangeOrigin::RemotePush` at transfer START, then at completion check via `consume_origin_or_default`. If it returns `RemotePush`, no user copy happened (origin was preserved). If it returns `LocalCapture` (default), the origin was consumed by a user clipboard change in between. Choose the simpler approach that fits the existing origin lifecycle. Document the chosen mechanism in the SUMMARY.

    d. **Clipboard write** (only if race not detected):
       - Build URI list from all file paths using `url::Url::from_file_path(&path)`
       - Build `SystemClipboardSnapshot` with `text/uri-list` representation
       - Set origin to `ClipboardChangeOrigin::RemotePush` with 2s TTL
       - Call `system_clipboard.write_snapshot(snapshot)` -- log warning on failure, don't fail transfer

    e. **Entry persistence (FCLIP-05)**: ALWAYS persist the clipboard entry regardless of clipboard race.
       Call `capture_clipboard_uc.execute_with_origin(snapshot.clone(), ClipboardChangeOrigin::RemotePush).await`.
       `CaptureClipboardUseCase::execute_with_origin` only skips for `LocalRestore`; `RemotePush` goes through full persistence (event, representation, entry creation). This creates the ClipboardEntry with text/uri-list representation in the database.
       If persistence fails, log error but don't fail the transfer.

    At the call site of `run_pairing_event_loop`, pass the new parameters:
    - `deps.clipboard.system_clipboard.clone()`
    - `deps.clipboard.clipboard_change_origin.clone()`
    - Build and pass a `CaptureClipboardUseCase` instance (or pass the deps needed to construct one inside the handler)

    **Important anti-patterns to avoid:**
    - Do NOT add `text/plain` representation -- decision says only `set_files()` path
    - Do NOT use `ClipboardChangeOrigin::LocalRestore` -- use `RemotePush` for inbound sync (LocalRestore would skip persistence)
    - Do NOT skip entry persistence when clipboard race is detected -- entry must be persisted so user can Copy from Dashboard
