  <action>
    1. 在 `src-tauri/crates/uc-tauri/src/bootstrap/runtime.rs` 中新增 `DaemonBootstrapOwnershipState`，最少记录：
       - `replacement_attempt: u8`
       - `spawned_child_pid: Option<u32>`
       - `last_incompatible_reason: Option<String>`
       它只服务 46.3 的 local daemon replacement，不承担其他 runtime 生命周期职责。
    2. 在 `src-tauri/crates/uc-tauri/src/bootstrap/run.rs` 中引入固定 replacement flow：
       - 如果 probe 为 `Absent`：直接 spawn 当前 daemon，并等待 compatible-ready。
       - 如果 probe 为 `Compatible`：直接加载 connection info。
       - 如果 probe 为 `Incompatible`：读取 PID file，对该 PID 执行本地终止，然后等待预期端点消失，再 spawn 当前 daemon，并再次等待 compatible-ready。
       - replacement 总预算固定为 1 次 incompatible replacement + 1 次新 daemon startup，不允许无限循环。
       - 旧 daemon 终止等待窗口固定为 `1500ms`，新 daemon compatible-ready 等待沿用 `HEALTH_CHECK_TIMEOUT = 8000ms`。
    3. 本地终止实现保持窄范围：
       - Unix：调用 `kill -TERM <pid>`
       - Windows：调用 `taskkill /PID <pid> /T /F`
       仅对 PID file 指向的 UniClipboard daemon 使用；不要做通用进程搜索或任意 host 管理。
    4. 在 `src-tauri/crates/uc-tauri/tests/daemon_bootstrap_contract.rs` 中新增回归测试，至少覆盖：
       - `daemon_bootstrap_replaces_incompatible_daemon_once`
       - `daemon_bootstrap_fails_after_bounded_replacement_attempt`
       - `daemon_bootstrap_does_not_replace_when_probe_is_absent`
