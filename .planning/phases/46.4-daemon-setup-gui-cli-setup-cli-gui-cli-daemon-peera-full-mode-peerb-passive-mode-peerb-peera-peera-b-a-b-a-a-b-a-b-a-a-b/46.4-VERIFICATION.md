### 1. Two-Terminal Happy Path

**Test:** Terminal A runs `UC_PROFILE=peerA UC_CLIPBOARD_MODE=full uniclipboard-cli setup host`; Terminal B runs `UC_PROFILE=peerB UC_CLIPBOARD_MODE=passive uniclipboard-cli setup join`; complete host accept and correct passphrase entry; then run `uniclipboard-cli devices` on both peers.
**Expected:** Both setup commands complete, and both `devices` outputs include the opposite peer.
**Why human:** Requires real daemon/network/discovery behavior and cross-process interactive terminals.

### 2. Wrong-Passphrase Retry

**Test:** During the same two-terminal flow, enter an incorrect passphrase once on peer B, then enter the correct passphrase.
**Expected:** Peer B prints `Passphrase rejected; retrying current join session`, remains in the same join flow, and succeeds after the second entry.
**Why human:** Full retry behavior depends on live peer interaction and terminal prompts, which were not exercised in this static verification pass.

### 3. Reset And Rerun

**Test:** After completion, run `uniclipboard-cli setup reset` on both peers and then repeat the host/join flow end-to-end.
**Expected:** Reset reports completion without stopping the daemon, and the second host/join round succeeds without manual file or DB cleanup.
**Why human:** Automated tests prove the cleanup internals and rerunnable state, but not the documented operator walkthrough against two live peer processes.
