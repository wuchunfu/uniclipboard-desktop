  <action>
    **Part A — Wire TaskRegistry into AppRuntime and wiring.rs:**

    1. Add a `task_registry: Arc<TaskRegistry>` field to `AppRuntime`. Create the registry in `AppRuntime::with_setup()` constructor. Expose via `pub fn task_registry(&self) -> &Arc<TaskRegistry>`.

    2. In `start_background_tasks()` (wiring.rs), accept `&Arc<TaskRegistry>` as a parameter. Replace all `async_runtime::spawn` calls for long-lived tasks with `registry.spawn("task_name", |token| async move { ... })`:

       - **Spool scanner + spooler + worker + janitor** (lines 1222-1280): The outer spawn becomes `registry.spawn("spool_scanner", ...)`. The nested spooler and worker spawns also go through registry. The janitor loop must `tokio::select!` on `token.cancelled()` vs `interval.tick()`.

       - **Space access completion loop** (line 1283): `registry.spawn("space_access_completion", ...)`. The inner `run_space_access_completion_loop` should exit when its channel closes (already does), but wrap with select on token for explicit cancellation.

       - **Clipboard receive loop** (lines 1297-1387): `registry.spawn("clipboard_receive", ...)`. Replace `tokio::signal::ctrl_c()` select branches with `token.cancelled()`. This is the most critical migration — the existing ctrl_c pattern is replaced by CancellationToken.

       - **Pairing event/action loops** (lines 1389-1407+): `registry.spawn("pairing_loops", ...)`. The nested action loop spawn (line 1396) must also go through registry.

    3. Short-lived per-event spawns (main.rs:613 UC protocol handler, runtime.rs:1002 outbound sync) remain as `async_runtime::spawn` but should check `registry.child_token().is_cancelled()` before starting new work. This is optional and can be deferred if it adds complexity.

    **Part B — Wire Tauri app exit hook in main.rs:**

    4. Change main.rs from `.run(tauri::generate_context!())` to `.build(tauri::generate_context!()).expect("..."); app.run(|app_handle, event| { ... })` pattern.

    5. In the `RunEvent::ExitRequested` branch: call `registry.token().cancel()` to signal all tasks. In the `RunEvent::Exit` branch: log "Application exiting". The `Arc<TaskRegistry>` is accessible via `app_handle.state::<Arc<TaskRegistry>>()` — register it with `.manage()` before `.build()`.

    6. Also store `platform_cmd_tx` in an accessible location (e.g., manage it as Tauri state) so that shutdown can send `PlatformCommand::Shutdown` to PlatformRuntime. If `PlatformCommand::Shutdown` variant exists, send it in the exit hook. If not, the CancellationToken approach is sufficient.

    **Part C — Pass TaskRegistry through call chain:**

    7. Update `start_background_tasks` call in main.rs setup closure to pass `runtime.task_registry()`.

    8. Ensure `cargo check` passes for the entire workspace after all changes.

    IMPORTANT: Do NOT add CancellationToken to any uc-core types or ports — it is a runtime/infra concern. Only uc-tauri and bootstrap code should reference tokio-util.
