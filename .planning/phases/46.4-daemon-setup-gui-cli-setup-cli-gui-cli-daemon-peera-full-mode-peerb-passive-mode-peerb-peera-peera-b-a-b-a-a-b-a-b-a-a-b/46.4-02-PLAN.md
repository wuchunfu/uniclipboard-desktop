<objective>
把 daemon setup transport 暴露为 CLI 的统一 `setup` 命令家族，形成可实际调试的 two-terminal `peerA` / `peerB` operator flow。

Purpose: 让 CLI 成为共享 setup 业务流的可交互入口，而不是另一套 setup 实现。
Output: `setup host`、`setup join`、`setup status` 命令，带 identity banner、daemon auto-probe/start、编号选 peer、隐藏口令输入和驻留式状态轮询。
</objective>

  <action>
    1. 在 `src-tauri/crates/uc-cli/src/main.rs` 中新增统一命令家族：
       - `uniclipboard-cli setup host`
       - `uniclipboard-cli setup join`
       - `uniclipboard-cli setup status`
       可选支持 `--json` 仅用于 `setup status`；`host` 和 `join` 默认走交互模式。
    2. 在 `src-tauri/crates/uc-cli/src/commands/setup.rs` 中实现共享 identity banner，输出固定字段：
       - `Profile: <profile>`
       - `Mode: <full|passive>`
       - `Device: <device name>`
       - `Peer ID: <peer id>`
       - `Session: <session id or ->`
    3. `setup host` 的具体行为固定为：
       - 调用本地 daemon helper 确保 daemon 可用
       - 调用 `start_setup_host()`
       - 若 daemon setup state / `nextStepHint` 为 `create-space-passphrase`，立即在当前终端隐藏输入两次相同 passphrase，并调用 `submit_setup_passphrase(passphrase)` 完成新空间初始化
       - 进入驻留轮询：持续读取 `get_setup_state()`
       - 当 `nextStepHint == "host-confirm-peer"` 或 state 为 `JoinSpaceConfirmPeer` 等价 host-confirm 阶段时，在当前终端提示 `accept / reject`
       - `accept` -> `confirm_setup_peer()`
       - `reject` -> `cancel_setup()`
       - setup 结束或取消时退出 host loop，但不停止 daemon
    4. `setup join` 的具体行为固定为：
       - 调用本地 daemon helper 确保 daemon 可用
       - 调用 `start_setup_join()`
       - 拉取当前 discovered peers（优先复用现有 daemon `/peers` 读模型），把可选 peer 以编号列表显示
       - 选择后调用 `select_setup_peer(peer_id)`
       - 当 daemon setup state 进入 `join-enter-passphrase` 或 `JoinSpaceInputPassphrase` 时，用隐藏输入采集 passphrase，调用 `submit_setup_passphrase(passphrase)`；该 transport 在 daemon 侧必须路由到 join-flow 的 `verify_passphrase(passphrase)`
       - 若 daemon state 回到 passphrase 输入且带 `PassphraseInvalidOrMismatch`，打印明确 retry 提示并继续留在当前命令会话，不强制用户重启命令
    5. `setup status` 必须输出 `state`、`sessionId`、`nextStepHint`、`profile`、`clipboardMode`、`deviceName`、`peerId`；human 模式复用 `output::print_result` 或等价统一打印规则。
    6. 命令错误语义固定为：
       - daemon 不可达 -> exit code `5`
       - 交互输入失败 / daemon 返回 invalid transition / internal failure -> exit code `1`
       - 正常完成 -> exit code `0`
  </action>
