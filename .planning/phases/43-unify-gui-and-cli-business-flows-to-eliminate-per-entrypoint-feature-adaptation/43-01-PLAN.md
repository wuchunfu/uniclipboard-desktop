## Current Problem

Each CLI command repeats this 4-step bootstrap sequence:

```rust
// Step 1: Build CLI context
let ctx = uc_bootstrap::build_cli_context_with_profile(profile)?;

// Step 2: Get storage paths
let storage_paths = uc_bootstrap::get_storage_paths(&ctx.config)?;

// Step 3: Build non-GUI runtime
let runtime = uc_bootstrap::build_non_gui_runtime(ctx.deps, storage_paths)?;

// Step 4: Create use cases
let usecases = uc_app::usecases::CoreUseCases::new(&runtime);
```

This pattern repeats in: clipboard.rs (3 functions), devices.rs (1 function), space_status.rs (1 function) = 5 repetitions.

## Target Solution

Add `build_cli_runtime()` in uc-bootstrap that returns `CoreRuntime` (NOT CoreUseCases - lifetime issue):

```rust
pub fn build_cli_runtime(
    log_profile: Option<uc_observability::LogProfile>,
) -> anyhow::Result<CoreRuntime> {
    let ctx = build_cli_context_with_profile(log_profile)?;
    let storage_paths = get_storage_paths(&ctx.config)?;
    let runtime = build_non_gui_runtime(ctx.deps, storage_paths)?;
    Ok(runtime)
}
```

CLI commands then become:

```rust
let runtime = uc_bootstrap::build_cli_runtime(Some(LogProfile::Cli))?;
let usecases = uc_app::usecases::CoreUseCases::new(&runtime);
```
