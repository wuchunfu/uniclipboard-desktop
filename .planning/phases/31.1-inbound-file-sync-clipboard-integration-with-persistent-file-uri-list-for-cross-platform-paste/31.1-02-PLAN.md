    - For file entries, populate `ClipboardFileItem`:
      ```typescript
      file: isFile ? {
        file_names: entry.preview.split('\n').filter(Boolean).map(uri => {
          try { return decodeURIComponent(new URL(uri).pathname.split('/').pop() || uri) }
          catch { return uri }
        }),
        file_sizes: [] // Size info not available from URI list; use entry.size_bytes as total
      } : null
      ```

    }

    - Add `markEntryStale` reducer action:
      ```typescript
      markEntryStale(state, action: PayloadAction<string>) {
        if (!state.staleEntryIds.includes(action.payload)) {
          state.staleEntryIds.push(action.payload)
        }
      }
      ```
    - Add `clearStaleEntries` reducer action (clears all, for use on data refresh):
      ```typescript
      clearStaleEntries(state) {
        state.staleEntryIds = []
      }
      ```
    - Export the actions: `markEntryStale`, `clearStaleEntries`

    **4. Wire copy in ClipboardContent.tsx:**
    - Import `copyFileToClipboard` from `@/api/clipboardItems`
    - Import `markEntryStale` from `@/store/slices/clipboardSlice`
    - Import `useAppDispatch` and `useAppSelector`
    - Select `staleEntryIds` from clipboard state
    - In the `onCopy` handler (or equivalent), when item type is `'file'`, call `copyFileToClipboard(itemId)` instead of the regular copy flow
    - Handle error: if the invoke rejects (error message will contain "file deleted" or similar), dispatch `markEntryStale(itemId)` and show toast notification with "file deleted" message (use existing toast/notification pattern)
    - Pass `isStale={staleEntryIds.includes(item.id)}` to `FileContextMenu` and `ClipboardItemRow` for each file item
