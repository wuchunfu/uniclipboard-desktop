  <action>
    1. 在 `src-tauri/crates/uc-tauri/src/bootstrap/daemon_lifecycle.rs` 中补齐退出清理能力，固定满足这些约束：
       - `begin_exit_cleanup()` 是 one-shot guard；第一次返回 true，后续重复进入返回 false
       - `shutdown_owned_daemon(timeout)` 只处理 live owned child；如果当前 owner 为空，必须快速 no-op 成功
       - 清理顺序优先优雅：先对 live child 发 TERM（复用现有本地 signal/helper 约束），等待退出；超时后再对该 live child 做 force kill + wait 兜底
       - cleanup 结束后必须清空 owner state，并允许 `finish_exit_cleanup()` 复位 guard
       - 不允许通过 pid file 决定“该不该杀”；pid file 只能作为 replacement 事实的辅助信息，不能成为 exit ownership 判据
    2. 在 `src-tauri/src/main.rs` 的 `RunEvent::ExitRequested` 路径实现真实退出 cleanup gate：
       - 只有当前存在 GUI-owned daemon 且 `begin_exit_cleanup()` 成功时，才 `api.prevent_exit()`
       - 然后异步执行 `shutdown_owned_daemon(...)`
       - cleanup 完成后再触发一次 `app.exit(0)` 放行真实退出
       - 第二次及后续 `ExitRequested` 不得重复 kill 同一 child，也不能进入递归死循环
    3. 明确保持 `on_window_event` 中的 `CloseRequested -> prevent_close + hide` 原样；不得把窗口关闭路径接到 daemon cleanup。
    4. 在日志里记录 `spawn_reason`、`pid`、`owned`/`non_owned`、`cleanup_result` 这类结构化字段，便于后续排查退出路径。
