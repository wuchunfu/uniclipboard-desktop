<objective>
Create FileTransferOrchestrator in uc-app and wire it into uc-bootstrap assembly, passed via BackgroundRuntimeDeps.

Purpose: Move all file transfer lifecycle orchestration (progress, completion, failure, timeout sweep, startup reconciliation) from standalone functions in uc-tauri into a proper orchestrator struct in uc-app, making it available to non-Tauri runtimes.

Key design: The orchestrator holds `emitter_cell: Arc<RwLock<Arc<dyn HostEventEmitterPort>>>` (the shared swappable cell) instead of a direct `Arc<dyn HostEventEmitterPort>`. This matches the existing `HostEventSetupPort` pattern and eliminates the emitter timing problem — the orchestrator can be constructed at wire time with the LoggingEventEmitter inside the cell, and automatically uses the TauriEventEmitter after the swap.

Output: FileTransferOrchestrator struct in uc-app, builder in uc-bootstrap assembly.rs, direct (non-Optional) field on BackgroundRuntimeDeps.
</objective>

**CRITICAL DESIGN: emitter_cell pattern (not direct Arc)**

The orchestrator holds the shared swappable emitter cell `Arc<RwLock<Arc<dyn HostEventEmitterPort>>>` — the same pattern used by `HostEventSetupPort` in `assembly.rs`. Every emit call reads the current emitter from the cell:

```rust
let emitter = self.emitter_cell.read().unwrap();
emitter.emit(HostEvent::Transfer(...))?;
```

This means:

- At wire time: cell contains `LoggingEventEmitter` → orchestrator constructed normally
- After Tauri setup: cell swapped to `TauriEventEmitter` → orchestrator automatically uses it
- No `Option`, no deferred construction, no `expect()`
