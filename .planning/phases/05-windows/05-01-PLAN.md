Replace the existing `read_image_windows()` function with two functions:

1. **`dib_to_png(dib_data: &[u8]) -> Result<Vec<u8>>`** -- A pure conversion function (no clipboard access) that takes raw CF_DIB data (BITMAPINFOHEADER + pixel data, NO BMP file header) and returns PNG-encoded bytes. This function is testable on all platforms because it only uses the `image` crate. Implementation:

   ```rust
   fn dib_to_png(dib_data: &[u8]) -> Result<Vec<u8>> {
       use image::codecs::bmp::BmpDecoder;
       use image::DynamicImage;
       use std::io::Cursor;

       let cursor = Cursor::new(dib_data);
       let decoder = BmpDecoder::new_without_file_header(cursor)
           .map_err(|e| anyhow::anyhow!("Failed to decode DIB: {}", e))?;
       let image = DynamicImage::from_decoder(decoder)
           .map_err(|e| anyhow::anyhow!("Failed to load DIB image: {}", e))?;

       let mut png_bytes = Vec::new();
       image.write_to(
           &mut Cursor::new(&mut png_bytes),
           image::ImageFormat::Png,
       ).map_err(|e| anyhow::anyhow!("Failed to encode PNG: {}", e))?;

       Ok(png_bytes)
   }
   ```

2. **`read_image_windows_as_png() -> Result<Vec<u8>>`** -- Windows-specific function (guarded by `#[cfg(target_os = "windows")]`) that reads CF_DIB from clipboard via clipboard-win and calls `dib_to_png`. Implementation:

   ```rust
   #[cfg(target_os = "windows")]
   fn read_image_windows_as_png() -> Result<Vec<u8>> {
       use clipboard_win::{formats, get_clipboard};

       let dib_data: Vec<u8> = get_clipboard(formats::RawData(formats::CF_DIB))
           .map_err(|e| anyhow::anyhow!("No DIB image on clipboard: {}", e))?;

       debug!(dib_size_bytes = dib_data.len(), "Read CF_DIB from Windows clipboard");
       dib_to_png(&dib_data)
   }
   ```

   Note: Uses `formats::RawData(formats::CF_DIB)` (not `formats::Bitmap`) because CF_DIB provides device-independent bitmap data. The `BmpDecoder::new_without_file_header` expects BITMAPINFOHEADER data directly (no BMP file header), which is exactly what CF_DIB provides.

3. Remove the old `read_image_windows()` function entirely. It is dead code with a critical bug (returns raw RGBA bytes, not PNG).

4. Add a `#[cfg(test)] mod tests` block at the bottom of `windows.rs` with the tests from the `<behavior>` block. The tests for `dib_to_png` do NOT need `#[cfg(target_os = "windows")]` because `dib_to_png` itself is platform-independent (only uses the `image` crate).    For the test data, construct a minimal valid BITMAPINFOHEADER (40 bytes) with a 2x2 32-bit BGRA image:

   ```rust
   fn make_dib_2x2_red() -> Vec<u8> {
       let mut data = Vec::new();
       // BITMAPINFOHEADER (40 bytes)
       data.extend_from_slice(&40u32.to_le_bytes()); // biSize
       data.extend_from_slice(&2i32.to_le_bytes());  // biWidth = 2
       // Negative height = top-down DIB (no vertical flip needed)
       data.extend_from_slice(&(-2i32).to_le_bytes()); // biHeight = -2 (top-down)
       data.extend_from_slice(&1u16.to_le_bytes());  // biPlanes = 1
       data.extend_from_slice(&32u16.to_le_bytes()); // biBitCount = 32
       data.extend_from_slice(&0u32.to_le_bytes());  // biCompression = BI_RGB
       data.extend_from_slice(&0u32.to_le_bytes());  // biSizeImage = 0
       data.extend_from_slice(&0i32.to_le_bytes());  // biXPelsPerMeter
       data.extend_from_slice(&0i32.to_le_bytes());  // biYPelsPerMeter
       data.extend_from_slice(&0u32.to_le_bytes());  // biClrUsed
       data.extend_from_slice(&0u32.to_le_bytes());  // biClrImportant
       // Pixel data: 4 red pixels in BGRA format
       for _ in 0..4 {
           data.extend_from_slice(&[0x00, 0x00, 0xFF, 0xFF]); // B=0, G=0, R=255, A=255
       }
       data
   }
   ```
