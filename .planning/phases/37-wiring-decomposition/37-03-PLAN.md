**Step 4: Keep invoke_handler! in main.rs (DECIDED — not moving command registration)**

`tauri::generate_handler!` is a compile-time macro that requires ALL commands in a single invocation — handlers cannot be merged at runtime. Three macOS-specific commands (`plugins::mac_rounded_corners::{enable_rounded_corners, enable_modern_window_style, reposition_traffic_lights}`) are defined in `src-tauri/src/plugins/`, a local module of the binary crate. These cannot be referenced from `uc_tauri` crate.

**Decision: The `invoke_handler!` block stays in main.rs.** This is the correct boundary — main.rs is the composition root that assembles commands from both uc_tauri and local plugin modules. No `build_invoke_handler()` function is created.

This does NOT conflict with the phase goal: the goal is splitting wiring.rs into assembly.rs + wiring.rs. Command registration ownership was a CONTEXT.md nice-to-have, not a success criterion. The invoke_handler block is already cleanly organized and does not contain business logic.
