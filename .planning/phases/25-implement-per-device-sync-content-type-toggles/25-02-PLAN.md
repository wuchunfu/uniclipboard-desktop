    2. Add `handleContentTypeToggle` callback (same pattern as `handleAutoSyncToggle`):
       ```typescript
       const handleContentTypeToggle = useCallback(
         (field: keyof ContentTypes) => {
           if (!settings) return
           dispatch(
             updateDeviceSyncSettings({
               peerId: deviceId,
               settings: {
                 ...settings,
                 content_types: {
                   ...settings.content_types,
                   [field]: !settings.content_types[field],
                 },
               },
             })
           )
         },
         [dispatch, deviceId, settings]
       )
       ```

    3. Add computed state for all-disabled warning:
       ```typescript
       const allContentTypesDisabled = settings
         ? Object.values(settings.content_types).every(v => !v)
         : false
       const showAllDisabledWarning = settings?.auto_sync && allContentTypesDisabled
       ```

    4. Update the content type toggle rendering in the `.map()`:
       - **Badge**: Use `t('devices.settings.badges.comingSoon')` for `coming_soon` status. Remove the `notEditable` badge entirely for `editable` status.
       - **Editable toggles (text, image)**:
         - When `auto_sync` is ON: fully interactive, use `cursor-pointer`, `onChange` calls `handleContentTypeToggle(field)`, not disabled
         - When `auto_sync` is OFF: visually grayed out (`opacity-50 cursor-not-allowed`), `disabled` attribute set, preserves checked value
         - Use same toggle styling as auto_sync toggle: `bg-muted peer-checked:bg-primary` for active state
       - **Coming soon toggles (file, link, code_snippet, rich_text)**:
         - Always disabled, `cursor-not-allowed opacity-50`
         - Show "Coming Soon" badge
         - Use muted styling: `peer-checked:bg-muted-foreground/40`
