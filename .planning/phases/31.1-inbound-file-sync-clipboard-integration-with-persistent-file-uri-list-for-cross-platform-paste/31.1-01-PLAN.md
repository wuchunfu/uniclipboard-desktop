Output: Backend clipboard integration for received files -- auto-write on transfer complete, manual copy command, clipboard entry persistence.

    **1. Create CopyFileToClipboardUseCase** (`copy_file_to_clipboard.rs`):
    - Struct fields: `local_clipboard: Arc<dyn SystemClipboardPort>`, `clipboard_change_origin: Arc<dyn ClipboardChangeOriginPort>`, `mode: ClipboardIntegrationMode`
    - `pub async fn execute(&self, file_paths: Vec<PathBuf>) -> Result<()>`:
      1. Check `self.mode.allow_os_write()` -- return error if disabled
      2. Validate each file path exists with `tokio::fs::metadata(&path).await` -- if ANY missing, return `Err(anyhow!("文件已删除"))`
      3. Build URI list: for each path, use `url::Url::from_file_path(&path)` to create `file://` URI (handles cross-platform differences). Join with `\n`
      4. Build `SystemClipboardSnapshot` with single representation: mime_type `text/uri-list`, format_id `files`, bytes = URI list string bytes
      5. Set origin: `self.clipboard_change_origin.set_next_origin(ClipboardChangeOrigin::RemotePush, Duration::from_secs(2)).await`
      6. Call `self.local_clipboard.write_snapshot(snapshot)` -- on error, consume origin back to LocalCapture (follow restore_clipboard_selection.rs pattern)
    - `pub async fn execute_from_entry_uris(&self, uri_list_content: &str) -> Result<()>`:
      1. Parse `file://` URIs from newline-separated string using `url::Url::parse(line)?.to_file_path()`
      2. Call `self.execute(parsed_paths).await`
    - Export from `mod.rs` with `pub use copy_file_to_clipboard::CopyFileToClipboardUseCase;`

    **2. Wire clipboard write in wiring.rs FileTransferCompleted handler**:
    - Add `system_clipboard: Arc<dyn SystemClipboardPort>` and `clipboard_change_origin: Arc<dyn ClipboardChangeOriginPort>` parameters to `run_pairing_event_loop`
    - Clone these Arc refs into the spawned task for FileTransferCompleted
    - After `handle_transfer_complete()` succeeds (Ok(result)):
      a. Build file URI with `url::Url::from_file_path(&result.file_path)`
      b. Build `SystemClipboardSnapshot` with `text/uri-list` representation
      c. Set origin to `ClipboardChangeOrigin::RemotePush` with 2s TTL
      d. Call `system_clipboard.write_snapshot(snapshot)`
      e. If write fails, log warning but don't fail the transfer
    - At the call site of `run_pairing_event_loop` (search for where it's invoked), pass `deps.clipboard.system_clipboard.clone()` and `deps.clipboard.clipboard_change_origin.clone()`

    **Important anti-patterns to avoid:**
    - Do NOT add `text/plain` representation -- decision says only `set_files()` path
    - Do NOT use `ClipboardChangeOrigin::LocalRestore` -- use `RemotePush` for inbound sync
    - Do NOT implement batch waiting in this task (multi-file batch is handled by frontend event batching from Phase 30)
