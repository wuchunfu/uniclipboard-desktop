  </read_first>
  <action>
    1. 在 `src-tauri/crates/uc-tauri/src/bootstrap/daemon_lifecycle.rs` 中新增 bounded cleanup 能力，例如：
       - `shutdown_owned_daemon(timeout: Duration, poll_interval: Duration) -> Result<bool, ...>`
       - 内部从 owned child 读取 pid，复用 `terminate_local_daemon_pid(pid)` 作为平台 kill primitive
       - 通过有界轮询确认 daemon endpoint absent 或 child 已退出
       - cleanup 成功后 `clear()`；失败则保留可观测日志，但不 panic
    2. 在 `src-tauri/src/main.rs` 的 `RunEvent::ExitRequested` 中：
       - 当 `GuiOwnedDaemonState::begin_exit_cleanup()` 返回 true 且当前确实存在 owned child 时，调用 `api.prevent_exit()`
       - 启动异步 cleanup 任务，完成后 `finish_exit_cleanup()`，再触发一次 `app_handle.exit(0)`
       - 当没有 owned child 或 cleanup 已在进行时，不重复进入双重 cleanup
    3. `RunEvent::Exit` 仍保留最终退出日志，但不要在这里新增第二套 daemon terminate 路径。
    4. 关闭主窗口 `CloseRequested` 逻辑保持 hide-to-tray；不要把 cleanup 接到这条路径上。
  </action>
  <verify>
    <automated>cd src-tauri && cargo test -p uc-tauri --test daemon_exit_cleanup -- --test-threads=1</automated>
  </verify>
  <acceptance_criteria>

    - `rg "RunEvent::ExitRequested|prevent_exit|app_handle.exit\\(0\\)|RunEvent::Exit" src-tauri/src/main.rs -n` 返回真实退出 wiring。
    - `rg "WindowEvent::CloseRequested" src-tauri/src/main.rs -n` 仍返回主窗口 hide-to-tray 路径，且未新增 daemon terminate 调用。
    - `cd src-tauri && cargo test -p uc-tauri --test daemon_exit_cleanup -- --test-threads=1` 退出码为 0。
  </acceptance_criteria>
  <done>
    - Tauri 真退出时会 best-effort 回收 GUI-owned daemon，不再留下孤儿进程。
    - 重复退出事件不会双杀或卡死。
  </done>
