  <behavior>
    - build_seq_layer returns None when UC_SEQ_URL env var is unset
    - build_seq_layer returns Some((layer, SeqGuard)) when UC_SEQ_URL is set
    - SeqLayer formats events using CLEFFormat and sends CLEF strings via mpsc channel
    - Background sender batches events (flush at 100 events or 2 seconds, whichever first)
    - Background sender POSTs to {url}/ingest/clef with Content-Type: application/vnd.serilog.clef
    - If UC_SEQ_API_KEY is set, X-Seq-ApiKey header is included
    - Channel uses try_send to avoid blocking tracing pipeline (silently drop if full)
    - SeqGuard drop sends shutdown signal and flushes remaining events with timeout
    - Seq errors are silently discarded (eprintln at most once for connection failures)
  </behavior>
  <action>
1. Create `src/seq/sender.rs`:
   - `pub struct SeqGuard` that holds a `tokio::sync::oneshot::Sender<()>` for shutdown signal and a `tokio::task::JoinHandle<()>` for the background task
   - `SeqGuard::drop()`: send shutdown signal, block on flush with 5-second timeout using `tokio::runtime::Handle::current().block_on()`
   - `pub(crate) async fn sender_loop(rx: mpsc::Receiver<String>, shutdown: oneshot::Receiver<()>, client: reqwest::Client, url: String, api_key: Option<String>)`:
     - Use `tokio::select!` with three branches: recv from mpsc, interval tick (2s), shutdown signal
     - Batch buffer: `Vec<String>` with capacity 100
     - On recv: push to batch, if batch.len() >= 100 -> flush
     - On tick: if batch not empty -> flush
     - On shutdown: flush remaining, return
   - `async fn flush_batch(client, url, api_key, batch)`: join with newlines, POST to `{url}/ingest/clef`, Content-Type `application/vnd.serilog.clef`, optional X-Seq-ApiKey header. Silently discard errors (`let _ = req.send().await`). Clear batch.

2. Create `src/seq/layer.rs`:
   - `pub(crate) struct SeqLayer` wrapping `tokio::sync::mpsc::Sender<String>`
   - Implement `tracing_subscriber::Layer<S>` for `SeqLayer`:
     - `on_event()`: format event to CLEF string using CLEFFormat, then `try_send()` on the mpsc channel. If channel full, silently drop.
     - For formatting: create a `String` buffer, use `CLEFFormat.format_event()` to write CLEF JSON into it, then send the string.
     - Note: Since `Layer::on_event` has access to the context, use `collect_span_fields` to gather span data, then serialize CLEF JSON manually (same logic as CLEFFormat but inline since we're in a Layer not FormatEvent). Alternatively, reuse the CLEFFormat serialization by writing to a String via `fmt::Write`.

3. Create `src/seq/mod.rs`:
   - `mod layer;` and `mod sender;`
   - `pub use sender::SeqGuard;`
   - `pub fn build_seq_layer<S>(profile: &LogProfile) -> Option<(impl tracing_subscriber::Layer<S> + Send + Sync, SeqGuard)>` where S: Subscriber + for<'a> LookupSpan<'a>:
     - Read `UC_SEQ_URL` env var. If unset, return None.
     - Read `UC_SEQ_API_KEY` env var (optional).
     - Create reqwest::Client (reuse across requests).
     - Create mpsc channel with buffer size 1024.
     - Create oneshot channel for shutdown.
     - Spawn background sender_loop task with tokio::spawn.
     - Create SeqLayer with the mpsc sender.
     - Apply profile.json_filter() to the layer.
     - Create SeqGuard with shutdown sender + join handle.
     - Return Some((filtered_layer, guard)).

4. Update `lib.rs`: add `pub mod seq;`, add `pub use seq::{build_seq_layer, SeqGuard};`

   - In seq/sender.rs: test_seq_guard_drop_flushes (create guard, drop it, verify shutdown signal sent)
   - Note: For HTTP tests, use unit tests that verify channel/batch behavior without actual HTTP calls. Integration testing with real Seq is a checkpoint in Plan 02.
