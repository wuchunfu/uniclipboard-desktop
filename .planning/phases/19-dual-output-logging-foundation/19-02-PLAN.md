  <action>
**Sentry integration approach (decisive):** Use the "layer builders" pattern. uc-observability exposes `build_console_layer()` and `build_json_layer()` as public functions. tracing.rs in uc-tauri calls these builders, creates the Sentry layer locally, and composes all three into `registry().with(sentry_layer).with(console_layer).with(json_layer).try_init()`. This keeps uc-observability free of any Sentry dependency. The existing `init_tracing_subscriber()` convenience function in uc-observability remains for standalone/test use (it calls the builders internally without Sentry).

1. **Update uc-observability to expose layer builders** (`src-tauri/crates/uc-observability/src/init.rs`):
   - Add two public functions alongside the existing `init_tracing_subscriber`:
     - `pub fn build_console_layer(profile: &LogProfile) -> impl Layer<S> + Send + Sync` — returns the pretty console fmt layer with per-layer EnvFilter, using the same logic already in `init_tracing_subscriber`.
     - `pub fn build_json_layer(logs_dir: &Path, profile: &LogProfile) -> anyhow::Result<(impl Layer<S> + Send + Sync, WorkerGuard)>` — returns the JSON file fmt layer with FlatJsonFormat, per-layer EnvFilter, daily rolling appender, and the WorkerGuard (caller is responsible for storing it).
   - Refactor `init_tracing_subscriber` to call these two builders internally, keeping it as a convenience wrapper.
   - Update `src-tauri/crates/uc-observability/src/lib.rs` to re-export the builder functions.

2. **Add uc-observability dependency to uc-tauri** (`src-tauri/crates/uc-tauri/Cargo.toml`):
   - Add: `uc-observability = { path = "../uc-observability" }`
   - Remove tracing-subscriber, tracing-appender from uc-tauri deps (they move to uc-observability). Keep tracing, sentry, sentry-tracing since Sentry init stays here.

3. **Rewrite `src-tauri/crates/uc-tauri/src/bootstrap/tracing.rs`**:
   - Remove all filter construction, layer building, file writer code — it's now in uc-observability.
   - Keep Sentry initialization (SENTRY_GUARD static, sentry::init, sentry_tracing::layer).
   - New `init_tracing_subscriber()` function (same no-arg signature for backward compat with main.rs):
     a. Resolve logs_dir: `DirsAppDirsAdapter::new().get_app_dirs()` -> `AppPaths::from_app_dirs()` -> `paths.logs_dir`.
     b. Resolve profile: `uc_observability::LogProfile::from_env()`.
     c. Init Sentry FIRST: check SENTRY_DSN env var, call `sentry::init()`, store guard in SENTRY_GUARD OnceLock, create `Option<sentry_tracing::layer()>`.
     d. Build layers from uc-observability: `let console = uc_observability::build_console_layer(&profile);` and `let (json, guard) = uc_observability::build_json_layer(&paths.logs_dir, &profile)?;` Store WorkerGuard in a static OnceLock.
     e. Compose: `tracing_subscriber::registry().with(sentry_layer).with(console).with(json).try_init()?;`
     f. Log the active profile and return Ok(()).

4. **Update `src-tauri/crates/uc-tauri/src/bootstrap/logging.rs`**:
   - **Remove** the `TargetKind::LogDir` target in production mode. The `uniclipboard.log` file is no longer produced.
   - **Keep** `TargetKind::Webview` for development (log::\* macros continue to browser DevTools).
   - **Keep** `TargetKind::Stdout` for production (log::\* still goes to terminal).
   - Update doc comments to reflect that file logging is now handled by tracing/uc-observability.

5. **Update `src-tauri/src/main.rs`** (if needed):
   - The call to `bootstrap_tracing::init_tracing_subscriber()` should still work since tracing.rs maintains the same public function signature. Verify no changes needed.

6. Verify the workspace compiles and tests pass.
