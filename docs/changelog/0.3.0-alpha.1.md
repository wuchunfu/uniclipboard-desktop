## 0.3.0-alpha.1 - 2026-03-12

### Features

- Add `uc-observability` with log profiles, flat JSON formatting, and tracing subscriber composition.
- Add Seq ingestion, bootstrap integration, and local Docker Compose support for log collection.
- Add `flow_id`/`origin_flow_id` correlation fields and stage spans across capture, outbound sync, inbound sync, and spool blob pipelines.
- Add `device_id` injection into Seq CLEF events for cross-device trace correlation.
- Add per-device sync settings across domain models, storage schema/migrations, repository mapping, use cases, and Tauri commands.
- Add frontend API functions and Redux thunks for per-device sync settings, and wire them into `DeviceSettingsPanel`.
- Add content-type classification and filtering in `uc-core`, with interactive content-type toggles in device settings.

### Fixes

- Remove the `allContentTypesDisabled` warning from device settings.
- Preserve Seq batches when send operations fail.
- Exit the Seq sender loop cleanly when the input channel closes.
- Update Seq signal JSON fields to Title Case for format compatibility.
- Add warning logs when inbound messages are missing `origin_flow_id`.
- Suppress noisy Hyper connection-pool debug logs in logging profiles.
- Complete `sync_settings` support in mocks and test implementations to keep behavior consistent.

### Refactor

- Centralize global `device_id` logging context handling.
- Rename the device ID resolver used by global logging context.

### Tests

- Add coverage for content-type classification and filtering with failing-first tests.
- Extend `DeviceSettingsPanel` tests for content-type toggle interaction.
- Add Nyquist validation coverage for tracing paths.
- Complete phase 25 UAT run with 4 passed checks and 1 skipped item.

### Docs

- Refresh architecture and project-structure documentation for the v0.3.0 observability milestone.
- Document cross-device tracing, Seq flow-query usage, and validation/audit strategy updates.
- Add and complete implementation plans for observability, per-device sync settings, and per-device content-type controls.
- Add a release-note generation command and update related release workflow documentation.
