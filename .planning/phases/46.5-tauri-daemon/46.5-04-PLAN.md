    1. 在 `src-tauri/crates/uc-tauri/src/commands/setup.rs` 中给全部 setup commands 增加 `daemon_connection: State<'_, DaemonConnectionState>`，并改成固定 daemon shell 映射：
       - `get_setup_state` -> `TauriDaemonSetupClient::get_setup_state()`
       - `start_new_space` -> `TauriDaemonSetupClient::start_new_space()`
       - `start_join_space` -> `TauriDaemonSetupClient::start_join_space()`

       并把响应里的 `state: serde_json::Value` 用 `serde_json::from_value::<SetupState>(...)` 反序列化回原有返回类型。
    2. `submit_passphrase(passphrase1, passphrase2)` 保持现有前端 contract：
       - 若 `passphrase1 != passphrase2`，直接返回 `Ok(SetupState::CreateSpaceInputPassphrase { error: Some(uc_core::setup::SetupError::PassphraseMismatch) })`
       - 若相等，则只把 `passphrase1` 传给 `TauriDaemonSetupClient::submit_passphrase(passphrase1)`，再把 ack.state 反序列化成 `SetupState`
    3. 在 `src-tauri/crates/uc-tauri/src/commands/pairing.rs` 中执行以下明确收口：
       - `get_p2p_peers` 继续使用 `TauriDaemonQueryClient::get_peers()`
       - `list_paired_devices` 改为 `TauriDaemonQueryClient::get_paired_devices()`
       - `get_paired_peers_with_status` 改为 `TauriDaemonQueryClient::get_paired_devices()`
       - `unpair_p2p_device` 改为 `TauriDaemonPairingClient::unpair_device(peer_id.clone())`
       - `get_local_peer_id` 与 `get_local_device_info` 保持本地读取
       - 删除 active command path 中对 `runtime.usecases().list_paired_devices()`、`runtime.usecases().get_p2p_peers_snapshot()`、`runtime.usecases().unpair_device()` 的直接调用
    4. 删除 `src-tauri/crates/uc-tauri/src/commands/pairing.rs` 里剩余 legacy `p2p-*` 兼容事件发射路径，尤其是 `p2p-command-error` emitter；命令失败只通过当前 typed command error 返回，不再保留第二条事件通道。
    5. `PairedPeer` 映射固定为：
       - `peer_id <- dto.peer_id`
       - `device_name <- dto.device_name`
       - `shared_secret <- vec![]`
       - `paired_at <- String::new()`
       - `last_seen <- dto.last_seen_at_ms` 的安全时间字符串映射，禁止 `unwrap()`
       - `last_known_addresses <- vec![]`
       - `connected <- dto.connected`
    6. 在 `src-tauri/crates/uc-tauri/tests/daemon_command_shell.rs` 先新增最小回归测试 `pairing_commands_use_daemon_queries_and_unpair_route`，用于锁定 Task 1 的 daemon query/unpair 路径与 legacy emitter 删除；这个测试必须在 Task 1 完成时就可运行。
