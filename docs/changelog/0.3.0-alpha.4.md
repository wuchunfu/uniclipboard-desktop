## 0.3.0-alpha.4 - 2026-03-13

### Features

- Add full keyboard shortcut settings with interactive key recording, conflict detection, and live updates.
- Add a storage management section in Settings with storage stats, cache cleanup, history cleanup, and open data directory actions.
- Replace clear-history browser confirm with an in-app dialog that supports keyboard confirmation shortcuts.
- Add a global sync paused banner in Devices and disable device sync controls when global sync is off.
- Add navigation support to open the Settings Sync tab directly from other flows.
- Add periodic timestamp refresh on the dashboard to keep relative time labels up to date.
- Introduce a new typography system with updated font families, scale tokens, and CJK support.
- Enforce global auto-sync policy in sync filtering so disabled global sync blocks outbound targets.

### Fixes

- Fix stale clipboard items flashing after clearing history from Settings by resetting Redux cache immediately.
- Fix retention policy updates to await backend results and roll back UI state on failures.
- Fix retention policy race conditions by using the latest optimistic rules snapshot.
- Fix storage stats loading to show explicit error and retry UI instead of silently failing.
- Fix clear-history pagination to avoid invalid limit errors on large datasets.
- Fix open data directory error classification by replacing fragile string matching with typed downcasting.
- Fix cache directory sizing to propagate filesystem errors instead of reporting misleading zero values.
- Fix storage path resolution to respect configured app paths and profile-aware locations.
- Fix DialogFooter spacing issues to remove extra bottom gap and preserve alignment.
- Fix Settings route state updates by using React Router navigation instead of direct history replacement.
- Fix Settings toggle initialization flash by deriving local state from loaded context values.
- Fix shortcut and i18n edge cases by removing hardcoded strings and improving update error handling.

### Refactor

- Refactor storage command handlers into dedicated use cases aligned with hexagonal architecture.
- Refactor clear cache filesystem operations behind a `CacheFsPort` adapter boundary.
- Refactor clear clipboard history command flow into a dedicated use case with structured results.
- Add typed storage IPC wrappers in `src/api/storage.ts` to centralize storage-related frontend calls.

### Tests

- Add unit tests for `GetStorageStats` and `OpenDataDirectory` use cases.
- Add global auto-sync policy tests covering override, fallback, resume, and mutation safety.
- Add timestamp refresh behavior tests with fake timers.

### Docs

- Add and complete planning and validation documentation for global sync master toggle and keyboard shortcuts settings.
- Add debug investigation notes for the stale dashboard flash after clear-history behavior.
- Improve phase summaries and plan records for quick timestamp refresh and related execution tracking.
