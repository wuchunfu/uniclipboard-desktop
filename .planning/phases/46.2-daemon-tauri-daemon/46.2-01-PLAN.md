  <action>
    1. 在 `src-tauri/crates/uc-daemon/src/pairing/host.rs` 中，确保 daemon pairing host 明确暴露 `initiate_pairing`、`accept_pairing`、`reject_pairing`、`cancel_pairing`、`register_gui_participant` 这些方法，并且所有 admission / busy / readiness / session-not-found 判定都留在 daemon 侧，不把任何业务决策推回 Tauri。
    2. 在 `src-tauri/crates/uc-daemon/src/pairing/host.rs` 中，统一错误 taxonomy，至少保留 `ActivePairingSessionExists`、`HostNotDiscoverable`、`NoLocalPairingParticipantReady`、`SessionNotFound`、`Internal`，并为关键路径加入 `tracing` 字段（至少包含 `session_id`、`peer_id`、`stage` 中的适用项）。
    3. 在 `src-tauri/crates/uc-daemon/src/api/routes.rs` 与 `src-tauri/crates/uc-daemon/src/api/server.rs` 中，固定 daemon pairing control surface：
       - `routes.rs` 负责 `/pairing/initiate`、`/pairing/accept`、`/pairing/reject`、`/pairing/cancel`、`/pairing/gui/lease`、`/pairing/sessions/:id/verify` 的 handler。
       - `server.rs` 只保留 `DaemonApiState`、共享装配和 `map_daemon_pairing_error`，不要把 pairing 业务逻辑塞回 server 层。
       - 错误映射保持稳定：busy / host_not_discoverable / no_local_participant / session_not_found / internal。
    4. 在 `src-tauri/crates/uc-daemon/src/api/ws.rs` 与 `src-tauri/crates/uc-daemon/src/state.rs` 中，固定 daemon realtime / snapshot contract：
       - websocket pairing topic 至少覆盖 `pairing.updated`、`pairing.verification_required`、`pairing.complete`、`pairing.failed` 这些稳定 type string。
       - `DaemonPairingSessionSnapshot` 保持 metadata-only，只允许 `session_id`、`peer_id`、`device_name`、`stage`、`updated_at_ms` 一类非敏感字段；不得把 `code`、`fingerprint`、`KeySlotFile` 或等价敏感信息写进 generic read model。
    5. 在 `src-tauri/crates/uc-daemon/tests/pairing_host.rs` 与 `src-tauri/crates/uc-daemon/src/state.rs` 的测试中补齐 regression 守卫：
       - `daemon_initiate_pairing_returns_busy_when_active_session_exists`
       - `daemon_initiate_pairing_returns_no_local_participant_when_not_ready`
       - `daemon_accept_reject_require_existing_session`
       - `pairing_session_snapshot_does_not_serialize_sensitive_fields`
       这些测试要直接绑定 daemon host / state contract，而不是依赖 Tauri 层间接覆盖。
