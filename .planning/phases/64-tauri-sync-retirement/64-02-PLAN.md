Wrap this entire outbound sync block in a Passive mode guard. In Passive mode, the daemon's ClipboardWatcherWorker will detect the OS clipboard write and handle outbound sync — so the command must NOT also sync.

Replace with:

```rust
// In Passive mode, daemon's ClipboardWatcherWorker handles outbound sync
// after detecting the OS clipboard write. Skip direct sync to avoid double-send.
if !matches!(
    runtime.clipboard_integration_mode(),
    ClipboardIntegrationMode::Passive
) {
    let outbound_sync_uc = runtime.usecases().sync_outbound_clipboard();
    match tokio::task::spawn_blocking(move || {
        outbound_sync_uc.execute(outbound_snapshot, uc_core::ClipboardChangeOrigin::LocalRestore, None, vec![])
    })
    .await
    {
        Ok(Ok(())) => {}
        Ok(Err(err)) => {
            tracing::warn!(error = %err, entry_id = %entry_id, "Restore outbound sync failed");
        }
        Err(err) => {
            tracing::warn!(error = %err, entry_id = %entry_id, "Restore outbound sync task join failed");
        }
    }
}
```

Also ensure that `let outbound_snapshot = snapshot.clone();` (line 573) is inside the guard or moved into it, so the clone is skipped in Passive mode too. If `outbound_snapshot` is used elsewhere, keep the clone outside.

Verify the `ClipboardIntegrationMode` import already exists at the top of the file (line 15: `use uc_app::usecases::clipboard::ClipboardIntegrationMode;`).
