# Plan 42.1-02: Migrate direct-mode CLI commands to shared bootstrap helper

## Goal

Replace every duplicated direct-mode bootstrap sequence in `uc-cli` command handlers with the shared helper introduced in Plan 01, while preserving all current CLI behavior.

## Tasks

<task id="1">
<name>Migrate devices.rs and space_status.rs to the shared helper</name>
<read_first>
- src-tauri/crates/uc-bootstrap/src/non_gui_runtime.rs
- src-tauri/crates/uc-bootstrap/src/lib.rs
- src-tauri/crates/uc-cli/src/commands/devices.rs
- src-tauri/crates/uc-cli/src/commands/space_status.rs
- src-tauri/crates/uc-cli/src/output.rs
- src-tauri/crates/uc-cli/src/exit_codes.rs
</read_first>
<action>
Update `src-tauri/crates/uc-cli/src/commands/devices.rs` and `src-tauri/crates/uc-cli/src/commands/space_status.rs` to import and call:

```rust
uc_bootstrap::build_cli_runtime_with_profile(profile)
```

Replace the current inline sequence that calls:

- `uc_bootstrap::build_cli_context_with_profile`
- `uc_bootstrap::get_storage_paths`
- `uc_bootstrap::build_non_gui_runtime`

In both files, preserve the existing command-specific behavior:

- `devices.rs` still constructs `CoreUseCases::new(&runtime)` and calls `list_paired_devices().execute().await`
- `space_status.rs` still calls `runtime.is_encryption_ready().await` and `runtime.encryption_state().await`
- both files still print the same stderr prefixes and return `exit_codes::EXIT_ERROR` on initialization or execution failure

When mapping helper failures, keep the current message prefix stable:

- `devices.rs`: `Error: failed to initialize CLI context:` or a single stable direct-bootstrap failure prefix applied consistently
- `space_status.rs`: same principle, with no behavior change to exit code handling

The helper call should appear exactly once per command function.
</action>
<acceptance_criteria>

- `src-tauri/crates/uc-cli/src/commands/devices.rs` contains `build_cli_runtime_with_profile`
- `src-tauri/crates/uc-cli/src/commands/space_status.rs` contains `build_cli_runtime_with_profile`
- `src-tauri/crates/uc-cli/src/commands/devices.rs` does not contain `build_cli_context_with_profile`
- `src-tauri/crates/uc-cli/src/commands/devices.rs` does not contain `get_storage_paths`
- `src-tauri/crates/uc-cli/src/commands/devices.rs` does not contain `build_non_gui_runtime`
- `src-tauri/crates/uc-cli/src/commands/space_status.rs` does not contain `build_cli_context_with_profile`
- `src-tauri/crates/uc-cli/src/commands/space_status.rs` does not contain `get_storage_paths`
- `src-tauri/crates/uc-cli/src/commands/space_status.rs` does not contain `build_non_gui_runtime`
  </acceptance_criteria>
  </task>

<action>
Update `src-tauri/crates/uc-cli/src/commands/clipboard.rs` so each direct-mode handler:
- `run_list`
- `run_get`
- `run_clear`

uses `uc_bootstrap::build_cli_runtime_with_profile(profile)` instead of repeating raw bootstrap calls.

Preserve the rest of the command body unchanged:

- `run_list` still calls `list_entry_projections().execute(limit, offset).await`
- `run_get` still constructs `EntryId::from_str(&id)` and calls `get_entry_detail().execute(&entry_id).await`
- `run_clear` still calls `clear_clipboard_history().execute().await`
- all three still pass results to `output::print_result`
- all three still return `exit_codes::EXIT_SUCCESS` on success and `EXIT_ERROR` on failure

Update `src-tauri/crates/uc-cli/tests/cli_smoke.rs` only as needed to preserve or strengthen regression coverage after the refactor. If you add assertions, keep them concrete and behavior-focused:

- `--help` still mentions `clipboard`
- empty clipboard list still prints `No clipboard entries found.`
- empty clear still prints `Cleared 0 clipboard entries.`
- missing entry still returns exit code `1`

Do not add new features, new flags, or unrelated cleanup.
</action>
