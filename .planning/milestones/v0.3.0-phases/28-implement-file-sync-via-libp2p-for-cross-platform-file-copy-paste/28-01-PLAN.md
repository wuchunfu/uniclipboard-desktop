# Plan 28-01: File transfer message types and filename validation

## Objective

Define the binary-encoded file transfer protocol messages and create a reusable filename validation module for security. These are pure domain types with no external dependencies.

## must_haves

- [ ] File transfer message enum with all 6 variants (Announce, Accept, Data, Complete, Cancel, Error)
- [ ] Binary encode/decode round-trip for all message variants
- [ ] Filename validation module rejecting all specified attack vectors
- [ ] Unit tests for both modules

## Tasks

<task id="28-01-01">
<title>Define file transfer message types with binary codec</title>
<description>
Create `uc-core/src/network/protocol/file_transfer.rs` with:

1. `FileTransferMessage` enum with variants:
   - `Announce { filename: String, file_size: u64, content_hash: String, batch_id: Option<String> }`
   - `Accept { transfer_id: String }`
   - `Data { chunk_index: u32, data: Vec<u8> }`
   - `Complete { content_hash: String }`
   - `Cancel { reason: String }`
   - `Error { code: u16, message: String }`

2. Binary codec using `std::io::Read/Write` following the pattern in `clipboard_payload_v3.rs`:
   - Each variant starts with a 1-byte discriminant tag
   - Strings: length-prefixed (u16 LE for short strings, u32 LE for data)
   - Safety limits: MAX_FILENAME_LEN=1024, MAX_CHUNK_SIZE=256*1024*1024, MAX_REASON_LEN=1024

3. Add `pub mod file_transfer;` to `uc-core/src/network/protocol/mod.rs` and re-export `FileTransferMessage`.

4. Unit tests:
   - Round-trip encode/decode for each variant
   - Reject oversized filename/data/reason
   - Invalid discriminant tag rejection

<task id="28-01-02">
<title>Create filename validation module</title>
<description>
Create `uc-core/src/security/filename_validation.rs` with:

1. `pub fn validate_filename(name: &str) -> Result<(), FilenameValidationError>` that rejects:
   - Empty or whitespace-only filenames
   - Filenames exceeding 255 bytes
   - Null bytes (`\0`) and control characters (`\x01-\x1F`)
   - Windows reserved names (CON, PRN, AUX, NUL, COM1-COM9, LPT1-LPT9, case-insensitive)
   - Leading dots (hidden files)
   - Unicode tricks: RTL override (`\u{202E}`), zero-width characters (`\u{200B}`, `\u{200C}`, `\u{200D}`, `\u{FEFF}`)
   - Path traversal: any `..` component, any path separator (`/`, `\`)
   - Basename extraction: if input contains separators, reject (caller must pass basename only)

2. `FilenameValidationError` enum with descriptive variants for each rejection reason.

3. Add `pub mod filename_validation;` to `uc-core/src/security/mod.rs` and re-export.

4. Unit tests covering every rejection rule with positive and negative cases.
