<objective>
Restructure the inbound business stream handler to use two-segment framing reads and stream-level V2 decoding, eliminating the read_to_end bottleneck. Update the clipboard channel to pass pre-decoded V2 plaintext to the use case layer.

Purpose: This plan delivers the core Phase 3 goal — V2 clipboard messages are decoded at the stream level using SyncIoBridge + spawn_blocking + ChunkedDecoder::decode_from, reducing peak memory from ~2-3x payload size to ~1x chunk size (256KB). The sync_inbound use case receives pre-decoded plaintext and no longer calls ChunkedDecoder itself.

Output: Restructured libp2p_network.rs inbound handler, updated clipboard_transport.rs port, updated sync_inbound.rs to use pre-decoded plaintext.
</objective>

**Part D: Restructure spawn_business_stream_handler**

The current handler at lines 855-928 does `stream.take(MAX).read_to_end()` then `handle_business_payload`. Replace the per-stream handler (the inner `tokio::spawn` block, lines 877-925) with the new two-segment reader:

```rust
tokio::spawn(async move {
    if check_business_allowed(&policy_resolver, &event_tx, &peer_id, ProtocolDirection::Inbound)
        .await
        .is_err()
    {
        return;
    }

    // Apply overall size guard on the stream
    let limited = stream.take(BUSINESS_PAYLOAD_MAX_BYTES + 1);

    // Convert libp2p stream to tokio AsyncRead
    use tokio_util::compat::FuturesAsyncReadCompatExt;
    let mut reader = limited.compat();

    let result = tokio::time::timeout(BUSINESS_READ_TIMEOUT, async {
        use tokio::io::AsyncReadExt;

        // Step 1: Read 4-byte JSON header length
        let mut len_buf = [0u8; 4];
        reader.read_exact(&mut len_buf).await
            .map_err(|e| format!("failed to read json header length: {e}"))?;
        let json_len = u32::from_le_bytes(len_buf) as usize;

        // Guard: cap JSON header size at 64KB
        const MAX_JSON_HEADER_SIZE: usize = 64 * 1024;
        if json_len > MAX_JSON_HEADER_SIZE {
            return Err(format!("json header too large: {json_len} > {MAX_JSON_HEADER_SIZE}"));
        }

        // Step 2: Read JSON header
        let mut json_buf = vec![0u8; json_len];
        reader.read_exact(&mut json_buf).await
            .map_err(|e| format!("failed to read json header: {e}"))?;

        let message = ProtocolMessage::from_bytes(&json_buf)
            .map_err(|e| format!("invalid protocol message: {e}"))?;

        match message {
            ProtocolMessage::Clipboard(msg)
                if msg.payload_version == ClipboardPayloadVersion::V2 =>
            {
                // Step 3: V2 streaming decode via spawn_blocking
                // Get master key from encryption session
                let master_key = match encryption_session.get_master_key().await {
                    Ok(k) => k,
                    Err(e) => {
                        return Err(format!("V2 inbound: encryption session not ready: {e}"));
                    }
                };

                let decode_result = tokio::task::spawn_blocking(move || {
                    use tokio_util::io::SyncIoBridge;
                    let sync_reader = SyncIoBridge::new(reader);
                    uc_infra::clipboard::ChunkedDecoder::decode_from(sync_reader, &master_key)
                })
                .await
                .map_err(|e| format!("V2 decode task panicked: {e}"))?;

                match decode_result {
                    Ok(plaintext) => {
                        Ok(ProcessedMessage::V2Clipboard(msg, plaintext))
                    }
                    Err(e) => {
                        Err(format!("V2 inbound: chunk decode failed: {e}"))
                    }
                }
            }
            other => {
                // V1 clipboard, DeviceAnnounce, Heartbeat, Pairing — no trailing payload
                Ok(ProcessedMessage::Standard(other))
            }
        }
    }).await;

    // Close stream (stream was moved into spawn_blocking for V2, so close is handled there)
    // For non-V2 paths, close the stream:
    // Note: for V2, the stream is consumed by spawn_blocking — SyncIoBridge drops it when done
    // For non-V2, we need to close. Since reader was moved into the async block, handle via result.

    match result {
        Ok(Ok(ProcessedMessage::V2Clipboard(msg, plaintext))) => {
            handle_v2_clipboard(caches, event_tx, clipboard_tx, peer_id, msg, plaintext).await;
        }
        Ok(Ok(ProcessedMessage::Standard(message))) => {
            handle_standard_message(caches, event_tx, clipboard_tx, peer_id, message).await;
        }
        Ok(Err(err)) => {
            warn!(peer_id = %peer_id, error = %err, "business stream processing failed");
        }
        Err(_) => {
            warn!(peer_id = %peer_id, "business stream read timed out");
        }
    }
});
```

**IMPORTANT design notes:**

1. **Stream ownership for V2:** When the stream is moved into `spawn_blocking` via `SyncIoBridge`, the blocking thread owns it. When `ChunkedDecoder::decode_from` finishes (or errors), the `SyncIoBridge` is dropped, which drops the underlying tokio reader, which drops the compat layer, which drops the `Take<libp2p::Stream>`. The libp2p stream close happens via Drop. If explicit close is needed, do it before moving into spawn_blocking — but since the receiver side just needs to finish reading, Drop is sufficient.

2. **EncryptionSessionPort threading:** The `encryption_session: Arc<dyn EncryptionSessionPort>` must be passed into `spawn_business_stream_handler`. This is a new parameter. Thread it from the network adapter's fields or the `run()` setup. Check where `spawn_business_stream_handler` is called and add the parameter.

3. **ProcessedMessage enum:** Define a local enum to distinguish V2 decoded results from standard messages:

```rust
enum ProcessedMessage {
    V2Clipboard(ClipboardMessage, Vec<u8>),
    Standard(ProtocolMessage),
}
```

5. **handle_v2_clipboard:** A small function that sends `(message, Some(plaintext))` through clipboard_tx and fires the ClipboardReceived event.
