**Daemon-backed `setup host|join|status` commands with identity banner, hidden passphrase input, peer selection, and profile-aware local daemon startup**

## Accomplishments

- Added authenticated CLI helpers for daemon `/setup/*` mutations plus `/peers` reads, reusing the existing profile-aware token/base-url resolution.
- Added a profile-aware local daemon helper that probes `/health`, spawns a sibling `uniclipboard-daemon` when needed, and polls until healthy without stopping the daemon afterward.
- Exposed `uniclipboard-cli setup host|join|status` with identity banners, numbered peer selection, hidden passphrase input, retry-on-invalid-passphrase behavior, and stable exit semantics.

## Decisions Made

- `setup status` reuses the shared JSON/human output convention by wrapping daemon setup state into a `Display + Serialize` view model instead of inventing a one-off printer.
- `setup host` treats initial daemon `completed` as the waiting state after local space creation, so the command remains attached for the intended two-terminal operator flow.
- `setup join` filters already paired peers out of numbered selection so the operator only sees viable join targets.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Serialized CLI smoke tests to stabilize `cargo test -p uc-cli`**

- **Found during:** Task 1 (扩展 CLI daemon client 与本地 daemon bootstrap helper)
- **Issue:** Existing `cli_smoke` tests launched multiple real CLI processes in parallel against shared local state, causing nondeterministic clipboard command failures and blocking required verification.
- **Fix:** Added a process-level mutex in `src-tauri/crates/uc-cli/tests/cli_smoke.rs` so the smoke tests run serially inside one test binary.
- **Files modified:** `src-tauri/crates/uc-cli/tests/cli_smoke.rs`

- **Found during:** Task 2 (实现 `setup host|join|status` 命令的交互式 operator flow)
- **Issue:** The repo had no existing terminal helper for masked passphrase input, which is required by both `setup host` and `setup join`.
