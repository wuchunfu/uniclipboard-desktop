       - `ProbeOutcome::Absent` -> spawn 当前 daemon，等 compatible-ready 成功后登记 `SpawnReason::Absent`
       - `ProbeOutcome::Incompatible` -> 完成 46.3 的 bounded replacement 后，新 spawn 的 daemon 等 compatible-ready 成功后登记 `SpawnReason::Replacement`
       - `ProbeOutcome::Compatible` -> 明确保持 owner empty，不登记任何 owned daemon
    2. 失败路径必须安全回滚：
       - 如果 spawn 成功但 compatible-ready 超时或失败，不能把半成品 child 留在 owner state 里
       - 如果 replacement 失败，owner state 必须保持空
    3. `src-tauri/src/main.rs` 中的 bootstrap 异步任务要把 managed `GuiOwnedDaemonState` 传给 `bootstrap_daemon_connection(...)`；本任务仍然不要实现 `ExitRequested` cleanup gate，那属于下一计划。
