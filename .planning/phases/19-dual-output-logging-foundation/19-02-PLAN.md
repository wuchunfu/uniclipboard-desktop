2. **Rewrite `src-tauri/crates/uc-tauri/src/bootstrap/tracing.rs`**:
   - Remove all filter construction, layer building, file writer code — it's now in uc-observability.
   - Keep Sentry initialization (SENTRY_GUARD static, sentry::init, sentry_tracing::layer).
   - New `init_tracing_subscriber()` function (same signature for backward compat with main.rs):
     a. Resolve logs_dir: `DirsAppDirsAdapter::new().get_app_dirs()` -> `AppPaths::from_app_dirs()` -> `paths.logs_dir`.
     b. Resolve profile: `uc_observability::LogProfile::from_env()`.
     c. Call `uc_observability::init_tracing_subscriber(&paths.logs_dir, profile)?;`
     d. Initialize Sentry (unchanged logic — check SENTRY_DSN, init guard, but NOTE: since the global subscriber is already set, sentry-tracing layer cannot be added to the registry after init. Two approaches:
     - **Option A (preferred)**: Pass an optional sentry layer into uc-observability's init function. This requires updating uc-observability's init signature to accept an optional boxed layer. However, this adds app-layer coupling to uc-observability.
     - **Option B**: Initialize Sentry BEFORE calling uc_observability init, then construct the sentry_tracing::layer() and pass it as a parameter. uc-observability's init function can accept `extra_layers: Vec<Box<dyn Layer<...>>>`.
     - **Option C (simplest, per CONTEXT.md discretion)**: Keep Sentry init in tracing.rs. Modify uc-observability's `init_tracing_subscriber` to accept an `Option<sentry_tracing::Layer>` or make it generic. OR — restructure so tracing.rs builds the layers from uc-observability components and composes them locally with Sentry.

     **Recommended approach**: Refactor tracing.rs to:
     1. Init Sentry first (get guard + optional sentry layer).
     2. Call `uc_observability::init_tracing_subscriber_with_extra_layer(logs_dir, profile, sentry_layer)` where the function accepts an `Option<impl Layer<S>>`.

     OR even simpler: have uc-observability expose `build_layers()` that returns the console and JSON layers, and tracing.rs does the final `registry().with(sentry).with(console).with(json).try_init()`. This keeps uc-observability free of sentry deps.

     **Decision**: Use the "build layers" approach. Change uc-observability's public API to:
     - `pub fn init_tracing_subscriber(logs_dir, profile)` — simple version, no sentry (for testing/standalone use)
     - `pub fn build_console_layer(profile) -> impl Layer` — returns configured console layer with filter
     - `pub fn build_json_layer(logs_dir, profile) -> Result<(impl Layer, WorkerGuard)>` — returns configured JSON layer with filter + guard

     Then tracing.rs composes: `registry().with(sentry_layer).with(console_layer).with(json_layer).try_init()`.

     **IMPORTANT**: If this changes the Plan 01 API design, adapt accordingly — the executor should update uc-observability's init.rs to also expose layer builders. The simple `init_tracing_subscriber` can still exist as a convenience that calls the builders internally.
     e. Return Ok(()).

   - Keep existing tests or update to match new thin wrapper.
