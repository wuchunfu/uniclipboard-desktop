# Phase 46.5 Plan 01: Pairing Runtime Ownership Summary

**Explicit pairing runtime ownership in uc-platform/uc-bootstrap with GUI-bound DisabledPairingTransport and daemon-only local pairing host**

## Performance

- **Duration:** 6 min
- **Started:** 2026-03-22T05:01:28Z
- **Completed:** 2026-03-22T05:06:52Z
- **Tasks:** 2
- **Files modified:** 5

## Accomplishments

- Added `PairingRuntimeOwner` and `DisabledPairingTransport` so GUI/CLI no longer pretend to own local pairing runtime.
- Gated `Libp2pNetworkAdapter` pairing service initialization, accept loop startup, and pairing protocol registration on runtime ownership.
- Routed GUI/CLI builders through shared bootstrap wiring with `ExternalDaemon`, while daemon keeps `CurrentProcess`.

## Task Commits

Each task was committed atomically:

1. **Task 1: 在 uc-platform 显式表达本地 pairing runtime owner 并为 GUI 提供 disabled pairing transport** - `c82132c2` (`impl`)
2. **Task 2: 在 uc-bootstrap 把 GUI/daemon 的 pairing ownership 显式传入共享 wiring** - `e0218df5` (`impl`)

## Files Created/Modified

- `src-tauri/crates/uc-platform/src/adapters/network.rs` - 定义 `PairingRuntimeOwner`、`DisabledPairingTransport` 与固定失败测试。
- `src-tauri/crates/uc-platform/src/adapters/libp2p_network.rs` - 为 `Libp2pNetworkAdapter` 注入 owner，并只在 `CurrentProcess` 下启动本地 pairing runtime。
- `src-tauri/crates/uc-platform/src/adapters/mod.rs` - 导出新的 owner/disabled transport 供 bootstrap 装配。
- `src-tauri/crates/uc-bootstrap/src/assembly.rs` - 将 owner 传入平台层装配，并根据 owner 切换 `NetworkPorts.pairing`。
- `src-tauri/crates/uc-bootstrap/src/builders.rs` - 固定 GUI/CLI 为 `ExternalDaemon`，daemon 为 `CurrentProcess`。

## Decisions Made

- 用 `PairingRuntimeOwner` 把“谁拥有本地 pairing runtime”提升为 composition root 输入，避免在 Tauri wiring 中继续靠忽略本地事件兜底。
- GUI/CLI 不再保留真实 `PairingTransportPort`，而是装配明确失败的 disabled adapter，这样任何残留本地 pairing 路径都会显式暴露。
