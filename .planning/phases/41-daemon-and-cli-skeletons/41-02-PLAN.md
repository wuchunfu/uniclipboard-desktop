pub async fn handle_request(
    request: &RpcRequest,
    state: &Arc<RwLock<RuntimeState>>,
) -> RpcResponse {
    match request.method.as_str() {
        "ping" => RpcResponse::success(
            request.id,
            serde_json::json!("pong"),
        ),
        "status" => handle_status(request.id, state).await,

        _ => RpcResponse::error(
            request.id,
            -32601,
            format!("Method not found: {}", request.method),
        ),
    }
}
```

Functions:

- `pub async fn check_or_remove_stale_socket(path: &Path) -> anyhow::Result<()>`:
  - If path doesn't exist, return Ok
  - Try `tokio::time::timeout(Duration::from_millis(200), UnixStream::connect(path))`
  - If connect succeeds: `anyhow::bail!("daemon already running at {:?}", path)`
  - If connect fails or times out: `warn!("removing stale socket at {:?}", path)`, `std::fs::remove_file(path)?`, return Ok

- `pub async fn run_rpc_server(socket_path: &Path, state: Arc<RwLock<RuntimeState>>, cancel: CancellationToken) -> anyhow::Result<()>`:
  - Call `check_or_remove_stale_socket(socket_path)?`
  - Bind `UnixListener::bind(socket_path)?`
  - `info!("RPC server listening on {:?}", socket_path)`
  - Loop with `tokio::select!`:
    - `result = listener.accept()` branch: spawn `handle_connection(stream, state.clone())`
    - `_ = cancel.cancelled()` branch: break
  - After loop: remove socket file with `std::fs::remove_file(socket_path).ok()`
  - `info!("RPC server stopped")`

- `async fn handle_connection(stream: UnixStream, state: Arc<RwLock<RuntimeState>>)`:
  - Split stream into reader/writer using `stream.into_split()` or use `BufReader`
  - Read one line (newline-delimited JSON)
  - Parse as `RpcRequest` via `serde_json::from_str`
  - If parse fails: write `RpcResponse::error(None, -32700, "Parse error")` and return
  - Call `handle_request(&request, &state).await`
  - Serialize response, append `\n`, write to stream
  - Flush and close

pub struct DaemonApp {

    socket_path: PathBuf,
    cancel: CancellationToken,

```

Methods:

- `pub fn new(workers: Vec<Box<dyn DaemonWorker>>, socket_path: PathBuf) -> Self`:
  - Creates `RuntimeState::new(workers)`, wraps in `Arc<RwLock<>>`
  - Creates `CancellationToken::new()`
  - Stores socket_path

- `pub async fn run(&self) -> anyhow::Result<()>`:
  1. Log startup: `info!("uniclipboard-daemon starting")`
  2. Start workers: iterate `self.state.write().await.workers()`, call `worker.start(self.cancel.child_token()).await?` for each. Use a child token so cancellation cascades
  3. Spawn RPC server: `tokio::spawn(run_rpc_server(&self.socket_path, self.state.clone(), self.cancel.child_token()))`
  4. Log running: `info!("uniclipboard-daemon running, RPC at {:?}", self.socket_path)`
  5. Wait for shutdown signal: call `wait_for_shutdown_signal().await`
  6. Log shutdown: `info!("shutdown signal received, stopping...")`
  7. Cancel: `self.cancel.cancel()`
  8. Stop workers in REVERSE order: iterate `self.state.write().await.workers()` in reverse, call `worker.stop().await`. Log each stop. Use `if let Err(e)` to log errors but continue stopping remaining workers
  9. Wait for RPC server task to finish (via JoinHandle or just give it a moment — the cancel token stops the accept loop)
  10. Remove socket file if it still exists: `std::fs::remove_file(&self.socket_path).ok()`
  11. `info!("uniclipboard-daemon stopped")`
  12. Return `Ok(())`

NOTE on worker start: the placeholder workers' `start()` awaits `cancel.cancelled()` and never returns until cancelled. So worker starts must be spawned as background tasks, NOT awaited inline. Spawn each worker start as `tokio::spawn(worker.start(token))`. Store JoinHandles if you want to await them during shutdown.

Actually, since `RuntimeState` owns the workers via `Vec<Box<dyn DaemonWorker>>`, and `start()` takes `&self`, the workers should be `Arc`-wrapped or the start calls spawned differently. Consider making `RuntimeState.workers` use `Vec<Arc<dyn DaemonWorker>>` instead of `Vec<Box<dyn DaemonWorker>>` — this allows cloning the Arc for spawn. Update state.rs accordingly if needed.

```rust
use uc_bootstrap::{build_daemon_app, build_non_gui_runtime};
use uc_daemon::app::DaemonApp;
use uc_daemon::workers::clipboard_watcher::ClipboardWatcherWorker;
use uc_daemon::workers::peer_discovery::PeerDiscoveryWorker;
use uc_daemon::worker::DaemonWorker;

fn main() -> anyhow::Result<()> {
    // build_daemon_app() calls build_core() which inits tracing + wires deps
    // MUST be called OUTSIDE tokio runtime (build_core is sync but creates internal tokio rt for GUI)
    // Actually build_daemon_app() does NOT create internal tokio rt (only build_gui_app does)
    // So it's safe to call from within #[tokio::main] OR before entering tokio
    let ctx = build_daemon_app()?;

    // Socket path: {data_dir}/uniclipboard-daemon.sock
    let socket_path = ctx.storage_paths.data_dir.join("uniclipboard-daemon.sock");

    // Create workers
    let workers: Vec<Box<dyn DaemonWorker>> = vec![
        Box::new(ClipboardWatcherWorker),
        Box::new(PeerDiscoveryWorker),
    ];

    // Create daemon app
    let daemon = DaemonApp::new(workers, socket_path);

    // Enter tokio runtime and run
    let rt = tokio::runtime::Builder::new_multi_thread()
        .enable_all()
        .build()?;
    rt.block_on(daemon.run())
}
```

Note: We use explicit `tokio::runtime::Builder` instead of `#[tokio::main]` because `build_daemon_app()` calls `build_core()` which is sync but calls `init_tracing_subscriber()` and `wire_dependencies()` — both sync. There's no conflict. However, using explicit runtime construction is consistent with the uc-bootstrap pattern (build_gui_app uses standalone runtime). If `build_daemon_app()` is confirmed to not use any internal `block_on()`, you MAY use `#[tokio::main]` and call `build_daemon_app()` inside it. Check the actual code and choose the safer approach.
