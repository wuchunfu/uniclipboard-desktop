- Outer retry loop with `subscribe_attempt` counter and backoff (replicate the exact backoff logic from `host.rs`: initial 250ms, max 30s, exponential factor)

  - `NetworkEvent::PeerLost(_peer_id)` -> identical snapshot fetch + emit `"peers"/"peers.changed"`
  - `NetworkEvent::PeerNameUpdated { peer_id, device_name }` -> emit `"peers"/"peers.name_updated"` with `PeerNameUpdatedPayload`

  - `NetworkEvent::PeerDisconnected(peer_id)` -> emit `"peers"/"peers.connection_changed"` with `PeerConnectionChangedPayload { peer_id, device_name: None, connected: false }`
  - `_ => {}` to ignore all other events
- Backoff on subscribe error: log warning with attempt count and retry_in_ms, sleep with cancellation check
