<objective>
Introduce two-segment wire framing for all outbound messages and update sync_outbound to produce length-prefixed bytes with raw V2 binary payload (no base64).

Purpose: This plan changes the sender wire format so that V2 binary payloads are sent as raw bytes after a length-prefixed JSON header, eliminating ~33% base64 overhead. All outbound messages (including non-clipboard) use the new length-prefix framing for uniformity. This is a prerequisite for Plan 03-02 which restructures the inbound reader to use stream-level parsing.

Output: Modified protocol_message.rs with framing methods, modified sync_outbound.rs using the new framing, updated tests.
</objective>

Add a `frame_to_bytes` method to `ProtocolMessage` in `protocol_message.rs`:

```rust
/// Produce two-segment wire format bytes.
///
/// All messages get a 4-byte LE length prefix before the JSON header.
/// If `trailing_payload` is `Some`, the raw bytes are appended after the JSON.
/// Used for V2 clipboard messages where the binary payload follows the JSON header.
pub fn frame_to_bytes(
    &self,
    trailing_payload: Option<&[u8]>,
) -> Result<Vec<u8>, serde_json::Error> {
    let json_bytes = serde_json::to_vec(self)?;
    let json_len = json_bytes.len() as u32;
    let trailing_len = trailing_payload.map_or(0, |p| p.len());
    let mut buf = Vec::with_capacity(4 + json_bytes.len() + trailing_len);
    buf.extend_from_slice(&json_len.to_le_bytes());
    buf.extend_from_slice(&json_bytes);
    if let Some(payload) = trailing_payload {
        buf.extend_from_slice(payload);
    }
    Ok(buf)
}
```

Keep `to_bytes` and `from_bytes` unchanged — they remain useful for internal use and tests.

fn parse_framed_v2(bytes: &[u8]) -> (ClipboardMessage, &[u8]) {
    let json_len = u32::from_le_bytes(bytes[0..4].try_into().unwrap()) as usize;
    let json_bytes = &bytes[4..4 + json_len];

    match ProtocolMessage::from_bytes(json_bytes).expect("decode protocol message") {
        ProtocolMessage::Clipboard(msg) => (msg, trailing),
        other => panic!("expected Clipboard, got {:?}", other),
    }
}
