    Rewrite `EncryptedBlobStore` to implement the V2 binary blob format with zstd compression.

    **Constants** (top of file):
    ```rust
    /// Magic bytes identifying a UniClipboard blob file ("UCBL")
    const BLOB_MAGIC: [u8; 4] = [0x55, 0x43, 0x42, 0x4C];
    /// Binary format version (v1 of the binary format, not to be confused with AAD v2)
    const BLOB_FORMAT_VERSION: u8 = 0x01;
    /// Header size: magic(4) + version(1) + nonce(24) = 29 bytes
    const BLOB_HEADER_SIZE: usize = 4 + 1 + 24;
    /// zstd compression level (3 = default, good speed/ratio balance)
    const ZSTD_LEVEL: i32 = 3;
    /// Maximum decompressed size to prevent zip bombs (500 MB)
    const MAX_DECOMPRESSED_SIZE: usize = 500 * 1024 * 1024;
    ```

    **Helper functions** (private, not methods):
    ```rust
    fn serialize_blob(nonce: &[u8; 24], ciphertext: &[u8]) -> Vec<u8> {
        let mut buf = Vec::with_capacity(BLOB_HEADER_SIZE + ciphertext.len());
        buf.extend_from_slice(&BLOB_MAGIC);
        buf.push(BLOB_FORMAT_VERSION);
        buf.extend_from_slice(nonce);
        buf.extend_from_slice(ciphertext);
        buf
    }

    fn parse_blob(data: &[u8]) -> Result<(&[u8; 24], &[u8])> {
        if data.len() < BLOB_HEADER_SIZE {
            return Err(anyhow::anyhow!("blob file truncated: {} bytes < {} header", data.len(), BLOB_HEADER_SIZE));
        }
        if data[0..4] != BLOB_MAGIC {
            return Err(anyhow::anyhow!("invalid blob magic bytes"));
        }
        if data[4] != BLOB_FORMAT_VERSION {
            return Err(anyhow::anyhow!("unsupported blob format version: {}", data[4]));
        }
        let nonce: &[u8; 24] = data[5..29].try_into()
            .map_err(|_| anyhow::anyhow!("nonce extraction failed"))?;
        Ok((nonce, &data[29..]))
    }
    ```

    **put method** — compress-then-encrypt pipeline:
    1. Get master key from session
    2. Compress plaintext: `zstd::bulk::compress(data, ZSTD_LEVEL)?`
    3. Build AAD: `aad::for_blob_v2(blob_id)`
    4. Encrypt compressed data via `self.encryption.encrypt_blob(...)`
    5. Extract nonce as `[u8; 24]` from `encrypted_blob.nonce`
    6. Serialize: `serialize_blob(&nonce, &encrypted_blob.ciphertext)`
    7. Write to inner store: `self.inner.put(blob_id, &binary_data).await?`
    8. Calculate on_disk_size = binary_data.len() as i64
    9. Return `Ok((path, Some(on_disk_size)))`
    10. Log with tracing: blob_id, plaintext_size, compressed_size, on_disk_size

    **get method** — decrypt-then-decompress pipeline:
    1. Read binary data from inner: `self.inner.get(blob_id).await?`
    2. Parse binary header: `parse_blob(&binary_data)?`
    3. Reconstruct `EncryptedBlob` struct from nonce + ciphertext (version V1, algo XChaCha20Poly1305, aad_fingerprint None)
    4. Get master key from session
    5. Build AAD: `aad::for_blob_v2(blob_id)`
    6. Decrypt: `self.encryption.decrypt_blob(&master_key, &encrypted_blob, &aad)?`
    7. Decompress: `zstd::bulk::decompress(&compressed, MAX_DECOMPRESSED_SIZE)?`
    8. Return plaintext

    1. **Runtime spool cleanup** — In `wiring.rs` (or the appropriate bootstrap location), add a one-time cleanup step that runs after Diesel migrations but before the app is fully operational. Near where `blob_store_dir` is created (around line 609):

       After `let blob_store_dir = config_dir.join("blobs");`, add logic to purge the directory contents if it exists. This handles orphaned blob files from the old JSON format that the Diesel migration (which only deletes DB records) cannot clean up.

       ```rust
       // Purge old blob files after V2 migration (old JSON format files are incompatible)
       // This is idempotent — if directory is empty or doesn't exist, it's a no-op
       if blob_store_dir.exists() {
           match std::fs::read_dir(&blob_store_dir) {
               Ok(entries) => {
                   let mut purged = 0u64;
                   for entry in entries.flatten() {
                       if entry.path().is_file() {
                           if let Err(e) = std::fs::remove_file(entry.path()) {
                               tracing::warn!(path = %entry.path().display(), error = %e, "Failed to purge old blob file");
                           } else {
                               purged += 1;
                           }
                       }
                   }
                   if purged > 0 {
                       tracing::info!(count = purged, "Purged old blob files (V2 format migration)");
                   }
               }
               Err(e) => {
                   tracing::warn!(error = %e, "Failed to read blob directory for cleanup");
               }
           }
       }
       ```

       NOTE: This cleanup should only run once — on subsequent starts the directory will be empty (or contain only V2 blobs). The migration handles the DB side; this handles the filesystem side. The cost is one readdir per startup which is negligible.

       IMPORTANT: The cleanup must happen BEFORE the `FilesystemBlobStore::new(blob_store_dir)` call (or immediately after — it doesn't matter since the store creates the directory lazily). Just ensure it's a one-shot cleanup that won't delete V2 blobs written in the same session. Since the cleanup runs synchronously during startup before any clipboard capture begins, this is safe.

       Consider: A more robust approach is to check a sentinel file (e.g., `.v2_migrated`) in the blob directory. If absent, purge all files and create the sentinel. This prevents re-purging on every startup:

       ```rust
       let sentinel = blob_store_dir.join(".v2_migrated");
       if blob_store_dir.exists() && !sentinel.exists() {
           // ... purge logic ...
           if let Err(e) = std::fs::File::create(&sentinel) {
               tracing::warn!(error = %e, "Failed to create V2 migration sentinel");
           }
       }
       ```
