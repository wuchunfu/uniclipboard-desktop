```rust
    }
}
```

Functions:

- `pub async fn check_or_remove_stale_socket(path: &Path) -> anyhow::Result<()>`:
  - If path doesn't exist, return Ok
  - Try `tokio::time::timeout(Duration::from_millis(500), UnixStream::connect(path))`
  - If connect succeeds: send a `ping` RPC request (`{"jsonrpc":"2.0","method":"ping","id":0}`) and check for `pong` response. If pong received: `anyhow::bail!("daemon already running at {:?}", path)`. If ping fails (invalid response, timeout): treat as stale
  - If connect fails or times out: `warn!("removing stale socket at {:?}", path)`, `std::fs::remove_file(path)?`, return Ok
  - This aligns with the CONTEXT.md locked decision requiring a full `ping` RPC check (not just TCP connect)

- `pub async fn run_rpc_accept_loop(listener: UnixListener, state: Arc<RwLock<RuntimeState>>, cancel: CancellationToken) -> anyhow::Result<()>` (takes already-bound listener — bind happens in DaemonApp::run() for fail-fast):
  - `info!("RPC server accepting connections")`
  - Loop with `tokio::select!`:
    - `result = listener.accept()` branch: spawn `handle_connection(stream, state.clone())`
    - `_ = cancel.cancelled()` branch: break
  - `info!("RPC server stopped")`

- `check_or_remove_stale_socket()` and `UnixListener::bind()` are called by `DaemonApp::run()` before this function is spawned. Socket file cleanup is also done by `DaemonApp::run()` during shutdown (after cancel).

- `async fn handle_connection(stream: UnixStream, state: Arc<RwLock<RuntimeState>>)`:
  - Split stream into reader/writer using `stream.into_split()` or use `BufReader`
  - Read one line (newline-delimited JSON)
  - Parse as `RpcRequest` via `serde_json::from_str`
  - If parse fails: write `RpcResponse::error(None, -32700, "Parse error")` and return
  - Call `handle_request(&request, &state).await`
  - Serialize response, append `\n`, write to stream
  - Flush and close

`DaemonApp` struct:

```rust
pub struct DaemonApp {
    workers: Vec<Arc<dyn DaemonWorker>>,  // DaemonApp owns workers (not RuntimeState)
    state: Arc<RwLock<RuntimeState>>,      // RuntimeState is snapshot-only
    socket_path: PathBuf,
    cancel: CancellationToken,
}
```

Workers use `Vec<Arc<dyn DaemonWorker>>` (not `Box`) to allow cloning for `tokio::spawn` `'static` requirement.

Methods:

- `pub fn new(workers: Vec<Arc<dyn DaemonWorker>>, socket_path: PathBuf) -> Self`:
  - Builds initial `Vec<WorkerStatus>` from workers (name + Healthy), creates `RuntimeState::new(initial_statuses)`, wraps in `Arc<RwLock<>>`
  - Creates `CancellationToken::new()`
  - Stores workers and socket_path

- `pub async fn run(&self) -> anyhow::Result<()>`:
  1. Log startup: `info!("uniclipboard-daemon starting")`
  2. **Bind RPC socket FIRST** (before starting workers — fail-fast): call `check_or_remove_stale_socket(&self.socket_path).await?`, then `let listener = UnixListener::bind(&self.socket_path)?`. If bind fails, we error before any worker starts. Do NOT spawn yet — the JoinHandle is created in step 5.
  3. Log running: `info!("uniclipboard-daemon running, RPC at {:?}", self.socket_path)`
  4. Start workers: for each worker, clone the Arc and spawn with explicit move:
     ```rust
     let mut worker_tasks = JoinSet::new();
     for worker in &self.workers {
         let w = Arc::clone(worker);
         let token = self.cancel.child_token();
         worker_tasks.spawn(async move { w.start(token).await });
     }
     ```
     Workers are spawned (not awaited inline) because placeholder workers block on `cancel.cancelled()`.
  5. Spawn accept loop and wait for EITHER shutdown signal, accept loop crash, OR worker crash:
     ```rust
     let rpc_handle = tokio::spawn(run_rpc_accept_loop(listener, self.state.clone(), self.cancel.child_token()));
     tokio::select! {
         _ = wait_for_shutdown_signal() => { info!("shutdown signal received"); },
         result = &mut rpc_handle => {
             warn!("RPC accept loop exited unexpectedly: {:?}", result);
         },
         Some(result) = worker_tasks.join_next() => {
             warn!("worker task exited unexpectedly: {:?}", result);
         },
     }
     ```
     This prevents the daemon from staying alive but deaf if accept loop or a worker crashes.
  6. Log shutdown: `info!("shutting down...")`
  7. Cancel: `self.cancel.cancel()`
  8. Drain worker tasks with timeout: `tokio::time::timeout(Duration::from_secs(5), async { while worker_tasks.join_next().await.is_some() {} }).await.ok();`
  9. Await RPC accept loop JoinHandle with timeout (5s). The accept loop internally drains its `JoinSet` of connection handlers before returning.
  10. Stop workers in REVERSE order: iterate `self.workers` in reverse, call `worker.stop().await`. Log each. Use `if let Err(e)` to log errors but continue
  11. Remove socket file: `if let Err(e) = std::fs::remove_file(&self.socket_path) { if e.kind() != std::io::ErrorKind::NotFound { warn!("failed to remove socket file: {}", e); } }`
  12. `info!("uniclipboard-daemon stopped")`
  13. Return `Ok(())`

- `async fn wait_for_shutdown_signal() -> anyhow::Result<()>` (returns Result, no expect/unwrap):
  ```rust
  #[cfg(unix)]
  {
      use tokio::signal::unix::{signal, SignalKind};
      let mut sigterm = signal(SignalKind::terminate())
          .map_err(|e| anyhow::anyhow!("failed to register SIGTERM handler: {}", e))?;
      tokio::select! {
          result = tokio::signal::ctrl_c() => {
              result.map_err(|e| anyhow::anyhow!("ctrl_c handler error: {}", e))?;
          },
          _ = sigterm.recv() => {},
      }
  }
  #[cfg(not(unix))]
  {
      tokio::signal::ctrl_c().await
          .map_err(|e| anyhow::anyhow!("ctrl_c handler error: {}", e))?;
  }
  Ok(())
  ```

RPC server should use `JoinSet` to track spawned connection handlers, draining them on shutdown:

```rust
}
```

```rust
use uc_bootstrap::{build_daemon_app, build_non_gui_runtime};
use uc_daemon::app::DaemonApp;
use uc_daemon::workers::clipboard_watcher::ClipboardWatcherWorker;
use uc_daemon::workers::peer_discovery::PeerDiscoveryWorker;
use uc_daemon::worker::DaemonWorker;

fn main() -> anyhow::Result<()> {
    // build_daemon_app() calls build_core() which inits tracing + wires deps
    // MUST be called OUTSIDE tokio runtime (build_core is sync but creates internal tokio rt for GUI)
    // Actually build_daemon_app() does NOT create internal tokio rt (only build_gui_app does)
    // So it's safe to call from within #[tokio::main] OR before entering tokio
    let ctx = build_daemon_app()?;

    // Socket path: {app_data_root}/uniclipboard-daemon.sock
    let socket_path = ctx.storage_paths.app_data_root.join("uniclipboard-daemon.sock");

    // Create workers (Arc-wrapped for spawn compatibility)
    let workers: Vec<Arc<dyn DaemonWorker>> = vec![
        Arc::new(ClipboardWatcherWorker),
        Arc::new(PeerDiscoveryWorker),
    ];

    // Create daemon app
    let daemon = DaemonApp::new(workers, socket_path);

    // Enter tokio runtime and run
    let rt = tokio::runtime::Builder::new_multi_thread()
        .enable_all()
        .build()?;
    rt.block_on(daemon.run())
}
```

Note: We use explicit `tokio::runtime::Builder` instead of `#[tokio::main]` because `build_daemon_app()` calls `build_core()` which is sync but calls `init_tracing_subscriber()` and `wire_dependencies()` — both sync. There's no conflict. However, using explicit runtime construction is consistent with the uc-bootstrap pattern (build_gui_app uses standalone runtime). If `build_daemon_app()` is confirmed to not use any internal `block_on()`, you MAY use `#[tokio::main]` and call `build_daemon_app()` inside it. Check the actual code and choose the safer approach.
