<objective>
Introduce the DTO mapping layer for command surfaces that currently leak domain models or
use raw string encoding as a workaround for typed serialization.

Purpose: Phase 11 CONTRACT-01 and CONTRACT-03. Commands must return explicit types that
the frontend can rely on. Three sites currently violate this: setup.rs double-encodes
SetupState to a JSON string, list_paired_devices returns Vec<PairedDevice> (core domain
model), and get_lifecycle_status returns a JSON-encoded string. Fix all three, coordinate
the frontend setup.ts change to remove the double-decode shim, and add serialization tests
for all command-layer DTOs.

Output: Updated models/mod.rs with LifecycleStatusDto and tests; updated setup.rs,
pairing.rs, lifecycle.rs with corrected return types; updated src/api/setup.ts removing
the JSON.parse decode layer.
</objective>
