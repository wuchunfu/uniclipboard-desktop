  <action>
In the `apply_sync_policy` method, after the `global_settings` load block (after line 76, before the `classify_snapshot` call at line 79), add an early-return guard:

```rust
// Global master toggle: if auto_sync is off, skip ALL outbound sync
if let Some(ref gs) = global_settings {
    if !gs.sync.auto_sync {
        info!("Global auto_sync disabled; returning empty peer list");
        return vec![];
    }
}
```

This must be BEFORE the `classify_snapshot` call and the per-device loop. This ensures:

- When global auto_sync is false, no sync happens to any device (hard override per user decision)
- Per-device settings are never touched or modified (they stay in storage unchanged)
- Settings load failure fallback is preserved (if global_settings is None, all peers proceed)
- When global is re-enabled, per-device filtering resumes as before (no code change needed -- the guard simply stops short-circuiting)

Do NOT modify any per-device settings storage when global toggle changes. The global toggle is an overlay/override, not an erase.
</action>
