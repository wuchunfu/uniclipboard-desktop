# Plan 46-01: Daemon Pairing Host Ownership And Runtime Projection

## Goal

Move the actual pairing host runtime into `uc-daemon` by giving the daemon bootstrap path ownership of the pairing orchestrator, its action receiver, and its network event loop, then project daemon-owned session state into `RuntimeState`.

<action>
Create `src-tauri/crates/uc-daemon/src/pairing/host.rs` with one daemon-local host type:

- `pub struct DaemonPairingHost`

Implement a constructor and run surface with explicit ownership of the migrated responsibilities:

- `pub fn new(...) -> Self`
- `pub async fn run(self, cancel: CancellationToken) -> anyhow::Result<()>`

`DaemonPairingHost` must own or receive:

- the daemon `Arc<CoreRuntime>`
- `Arc<PairingOrchestrator>`
- `mpsc::Receiver<PairingAction>`
- `Arc<RwLock<RuntimeState>>`
- `Arc<SpaceAccessOrchestrator>`
- `Arc<dyn KeySlotStore>`

Keep the long-lived responsibilities in daemon code, not in Tauri:

- pairing action execution that opens, writes, and closes pairing sessions
- retrying subscription to pairing network events
- translation of pairing domain events into daemon session projection updates
- explicit discovery-visibility control with headless/ordinary-CLI defaulting to non-discoverable
- explicit single-active-session enforcement
- explicit local-participant-readiness enforcement for inbound requests

`DaemonPairingHost` must model discovery visibility separately from participant readiness:

- pure daemon / ordinary CLI startup defaults to `discoverable = false`
- GUI-managed clients may set `discoverable = true` during startup
- CLI pairing mode must explicitly opt into discoverability before the host appears in peer discovery
- inbound pairing must never create a session while the host is non-discoverable

Define one concrete host-side error taxonomy for the gate decisions so later API routes can map them without rediscovery:

- `ActivePairingSessionExists`
- `NoLocalPairingParticipantReady`

Create `src-tauri/crates/uc-daemon/src/pairing/session_projection.rs` with these exact helpers:

- `upsert_pairing_snapshot(...)`
- `mark_pairing_session_terminal(...)`
- `remove_expired_pairing_session(...)`

These helpers must write `DaemonPairingSessionSnapshot { session_id, peer_id, device_name, state, updated_at_ms }` into `RuntimeState`. They must never store short codes, fingerprints, `KeySlotFile`, or raw challenge bytes.

Update `src-tauri/crates/uc-daemon/src/app.rs` so `DaemonApp::new(...)` accepts the new pairing-runtime dependencies and `DaemonApp::run()` starts one `DaemonPairingHost` task alongside HTTP, RPC, and workers. The host task must stay alive even if a Tauri/webview client disconnects. Session lifetime ends only on timeout or terminal result.

Add `src-tauri/crates/uc-daemon/tests/pairing_host.rs` with concrete coverage for:

- `daemon_pairing_host_enforces_single_active_session`

- `daemon_pairing_host_survives_client_disconnect`
