# Phase 46.1 Plan 04: Frontend Contract Cutover To daemon://realtime Summary

**Frontend and Tauri adapter runtime paths now consume one `daemon://realtime` envelope instead of domain-specific event names**

## Performance

- **Duration:** 9 min
- **Started:** 2026-03-20T22:56:26+0800
- **Completed:** 2026-03-20T23:05:50+0800
- **Tasks:** 2
- **Files modified:** 9

## Accomplishments

- Added `src/api/realtime.ts` with the shared `DAEMON_REALTIME_EVENT` constant and listener helper.
- Migrated pairing, peers, setup, and space-access frontend listeners to route through the shared realtime envelope instead of active `p2p-*` or `setup-state-changed` listeners.
- Updated the Tauri adapter so setup state changes and space-access completions also emit on `daemon://realtime`, keeping the runtime contract single-channel.
- Added dedicated frontend contract tests for the realtime helper, `useDeviceDiscovery`, and `PairingNotificationProvider`.

## Task Commits

Each task was committed atomically:

1. **Task 1: Create failing frontend contract and consumer tests for the unified realtime channel** - `674f1a30` (test)
2. **Task 2: Switch Tauri emission and frontend listeners to the daemon://realtime envelope** - `4129500a` (impl)

## Files Created/Modified

- `src/api/realtime.ts` - Shared frontend listener helper for the daemon realtime channel.
- `src/api/p2p.ts` - Realtime wrapper APIs now translate shared envelopes into pairing/peer callbacks.
- `src/api/setup.ts` - Setup state listener now consumes shared setup envelopes.
- `src/hooks/useDeviceDiscovery.ts` - Device discovery hook now derives state directly from peers realtime envelopes.
- `src/components/PairingNotificationProvider.tsx` - Pairing UX now routes request/verification/space-access events through the shared realtime helper.
- `src/api/__tests__/p2p-realtime-contract.test.ts` - Contract coverage for the single event name and camelCase payload handling.
- `src/hooks/__tests__/useDeviceDiscovery.realtime.test.ts` - Hook coverage for peers.changed and peers.nameUpdated.
- `src/components/__tests__/PairingNotificationProvider.realtime.test.tsx` - Unified pairing/setup routing coverage.
- `src-tauri/crates/uc-tauri/src/adapters/host_event_emitter.rs` - Setup and space-access host events now serialize onto the shared realtime channel.

## Decisions Made

- Kept existing frontend-facing wrapper function names (`onP2PPairingVerification`, `onSetupStateChanged`, etc.) but rewired them to the shared helper so downstream components could migrate incrementally without keeping dual Tauri listeners alive.
- Ran frontend verification through `bun run test -- ...` because the repo’s `package.json` test entrypoint is `vitest`; plain `bun test` does not honor the same jsdom/mock environment.
