  <action>
    1. 新增 `src/api/startup.ts`，固定导出：
       - `getStartupStatus()`
       - `retryStartupDaemon()`
       它们分别调用 Tauri command `get_startup_status` 与 `retry_startup_daemon`，并在 API 层把返回值约束成 `StartupStatusSnapshot`。
    2. 新增 `src/hooks/useStartupStatus.ts`，实现固定 polling 规则：
       - 当 `phase === 'bootstrapping'` 或 `isRetrying === true` 时，每 `1000ms` 轮询一次 `getStartupStatus()`
       - 当 `phase === 'failed'` 且 `nextRetryAtMs != null` 时，每 `1000ms` 轮询一次，只负责读取 shell 侧自动恢复进度，不在前端自行触发 retry
       - 当 `phase === 'failed'` 且 `nextRetryAtMs == null` 且 `isRetrying === false` 时，停止自动 polling，等待用户手动重试
       - 当页面触发手动重试时，调用 `retryStartupDaemon()`，并在 `isRetrying` 为 true 时禁止重复点击
       - 一旦轮询看到 `phase === 'ready'`，立即停止轮询并把状态交还给上层 App gate
    3. 在 `src/api/__tests__/startup.test.ts` 中覆盖：
       - `getStartupStatus` 调用正确 command 名与返回映射
       - `retryStartupDaemon` 调用正确 command 名
       - API 层保留 `failureCode/message/canRetry/isRetrying/nextRetryAtMs` 字段
  </action>
  <verify>

  </verify>

    - `rg "get_startup_status|retry_startup_daemon" src/api/startup.ts src/api/__tests__/startup.test.ts -n` 返回固定 command 名。
    - `rg "nextRetryAtMs|isRetrying|1000" src/hooks/useStartupStatus.ts src/api/startup.ts src/api/__tests__/startup.test.ts -n` 返回 bounded polling 条件与字段映射。
