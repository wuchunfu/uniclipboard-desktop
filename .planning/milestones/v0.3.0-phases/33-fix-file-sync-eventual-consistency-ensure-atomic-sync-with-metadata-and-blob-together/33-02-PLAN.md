    1. Add a migration that upgrades `file_transfer` for metadata-first persistence. The resulting table must include:
       - `entry_id TEXT NOT NULL`
       - `failure_reason TEXT NULL`
       - `file_size BIGINT NULL`
       - `content_hash TEXT NULL`
       - existing `cached_path`, `status`, `source_device`, timestamps
       - `updated_at_ms` retained as the durable liveness timestamp used by timeout sweeps

    2. Preserve existing rows during migration. If SQLite requires table rebuild semantics, do the standard create-copy-drop-rename flow instead of relying on unsupported column alterations.

    3. Add indexes that make the read patterns cheap:
       - by `entry_id`
       - by `status`
       - optionally by `(entry_id, status)` if the repository benefits from it

    4. Update Diesel schema/model files to match the new shape exactly.

    5. Do NOT add a FOREIGN KEY constraint on `entry_id`. SQLite FK enforcement is off by default (`PRAGMA foreign_keys` is not enabled), and `clipboard_entry` may be managed by a different crate's migration. Cross-table FK increases coupling without practical benefit here.

    6. Mark `down.sql` as destructive: rolling back will lose `entry_id` and `failure_reason` data. Add a comment in `down.sql` explaining this is a schema reshape (not an incremental column add) and data loss is expected on rollback.

    7. Keep this plan infra-only. Do not wire the adapter into runtime or Tauri yet.

    1. Implement `FileTransferRepositoryPort` in a dedicated repository file. Required operations:
       - seed pending rows for an entry
       - backfill announce metadata and path
       - mark transferring
       - refresh in-flight activity by bumping `updated_at_ms`
       - mark completed
       - mark failed with reason
       - list or sweep timed-out rows using status + `updated_at_ms`
       - bulk fail in-flight rows on startup
       - compute entry-level aggregate summary

    2. Make terminal transitions idempotent where practical. A repeated `mark_failed` or `mark_completed` should not produce duplicate rows or inconsistent state.

    3. Add repository tests for:
       - metadata seed creates pending rows with `entry_id`
       - announce backfill fills `file_size`, `content_hash`, and `cached_path`
       - failed rows retain `failure_reason`
       - aggregate summary returns `failed` over `transferring`/`pending`
       - timeout query returns `pending` rows older than 60 seconds and `transferring` rows older than 5 minutes
       - startup reconciliation only touches `pending` and `transferring`

    4. Tests must use in-memory SQLite (consistent with existing `uc-infra` test patterns) initialized via `run_migrations()`.

    5. Register the repository module so later plans can wire it without infra reshuffling.
