<objective>
Create the V3 binary payload codec in uc-core and the V3 chunked encoder/decoder with zstd compression in uc-infra.

Purpose: These are the foundational building blocks for the V3 wire protocol. The binary codec eliminates JSON+base64 overhead (~33% savings), and zstd compression before encryption provides ~60%+ reduction on image payloads. Both must be fully tested before use cases are rewired.

Output: Two new/modified source files with complete encode/decode implementations and comprehensive unit tests.
</objective>

  <files>src-tauri/crates/uc-infra/src/clipboard/chunked_transfer.rs</files>
  <behavior>
    - V3 round-trip: small payload (< 8KB, no compression) encode/decode produces identical plaintext
    - V3 round-trip: large payload (> 8KB, with compression) encode/decode produces identical plaintext
    - V3 round-trip: 10MB payload encode/decode produces identical plaintext
    - V3 header: starts with V3_MAGIC [0x55, 0x43, 0x33, 0x00], total header is 37 bytes
    - V3 compression flag: payload <= 8KB has compression_algo=0 in header; payload > 8KB has compression_algo=1
    - V3 uncompressed_len: header field matches original plaintext length
    - V3 total_plaintext_len: equals compressed size (when compressed) or original size (when not)
    - V3 tampered ciphertext returns DecryptFailed
    - V3 wrong key returns DecryptFailed
    - V3 invalid magic returns InvalidMagic
    - V3 adapter: TransferPayloadEncryptorAdapter/DecryptorAdapter round-trip through V3 format
    - Existing V2 tests still pass (V2 code not removed yet)
  </behavior>
  <action>
Add V3 encoder/decoder to `chunked_transfer.rs` alongside existing V2 code (V2 stays until Plan 02):

1. Add constants: `V3_MAGIC: [u8; 4] = [0x55, 0x43, 0x33, 0x00]`, `V3_HEADER_SIZE: usize = 37`, `COMPRESSION_THRESHOLD: usize = 8 * 1024`, `ZSTD_LEVEL: i32 = 3`
2. Add error variants to `ChunkedTransferError`: `CompressionFailed { source: String }`, `DecompressionFailed { source: String }`, `InvalidCompressionAlgo { algo: u8 }`. Update the `From<ChunkedTransferError> for TransferCryptoError` impl to map these.
3. Create `ChunkedEncoderV3` with `encode_to<W: Write>` that:
   - Takes `(writer, master_key, transfer_id, plaintext, compression_algo: u8, uncompressed_len: u32)` — compression decision is made by the adapter caller
   - Writes 37-byte V3 header: magic(4) + compression_algo(1) + uncompressed_len(4) + transfer_id(16) + total_chunks(4) + chunk_size_hint(4) + total_plaintext_len(4)
   - Note: `total_plaintext_len` is the length of `plaintext` parameter (compressed size when compression active)
   - Chunk AEAD loop same as V2 (reuse `derive_chunk_nonce` and `aad::for_chunk_transfer`)
4. Create `ChunkedDecoderV3` with `decode_from<R: Read>` that:
   - Reads 37-byte header, validates V3_MAGIC
   - Extracts compression_algo, uncompressed_len, transfer_id, total_chunks, chunk_size_hint, total_plaintext_len
   - Decrypts chunks same as V2
   - After full decrypt: if compression_algo == 1, decompress with `zstd::bulk::decompress(&plaintext, uncompressed_len as usize)`. If compression_algo == 0, return plaintext directly. If other value, return `InvalidCompressionAlgo`.
5. Update `TransferPayloadEncryptorAdapter::encrypt` to use V3:
   - Decide compression: if `plaintext.len() > COMPRESSION_THRESHOLD`, compress with zstd level 3, set algo=1, uncompressed_len=plaintext.len(). Else algo=0, uncompressed_len=plaintext.len(), pass plaintext through.
   - Call `ChunkedEncoderV3::encode_to` with compressed (or uncompressed) data
6. Update `TransferPayloadDecryptorAdapter::decrypt` to detect format by magic:
   - Read first 4 bytes: if V3_MAGIC, use `ChunkedDecoderV3::decode_from`; if V2_MAGIC, use `ChunkedDecoder::decode_from` (temporary, until V2 removed in Plan 02)
   - This enables a transition period where both formats decode correctly
7. Keep all existing V2 tests passing. Add new V3 tests in the same `#[cfg(test)]` module.
