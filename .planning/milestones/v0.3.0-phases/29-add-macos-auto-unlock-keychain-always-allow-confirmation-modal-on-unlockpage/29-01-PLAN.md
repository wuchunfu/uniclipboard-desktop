<objective>
Create the backend verification mechanism for macOS Keychain "Always Allow" permission.

Purpose: The frontend needs a dedicated Tauri command that checks whether Keychain access succeeds silently (meaning "Always Allow" was granted) without performing the full unlock flow. This is a lighter operation than auto_unlock_encryption_session -- it only calls `load_kek()` and reports whether the call succeeded without triggering a user prompt.

Output: A new `VerifyKeychainAccess` use case, a `verify_keychain_access` Tauri command, and the wiring between them.
</objective>

   - `Unexpected(String)` -- unexpected error from keyring

2. Define `VerifyKeychainAccess` struct with only two ports (lighter than AutoUnlock):
   - `key_scope: Arc<dyn KeyScopePort>`
   - `key_material: Arc<dyn KeyMaterialPort>`

3. Implement `new()` and `from_ports()` constructors (follow AutoUnlockEncryptionSession pattern).

4. Implement `execute(&self) -> Result<bool, VerifyKeychainError>`:
   - Get current scope via `self.key_scope.current_scope().await`
   - Call `self.key_material.load_kek(&scope).await`
   - `Ok(_)` -> return `Ok(true)` (Keychain access succeeded silently = Always Allow granted)
   - `Err(EncryptionError::PermissionDenied)` -> return `Ok(false)` (user denied or didn't grant)
   - `Err(EncryptionError::KeyNotFound)` -> return `Err(VerifyKeychainError::KekNotFound)`
   - `Err(EncryptionError::KeyringError(_))` -> return `Ok(false)` (treat any keyring error as not granted)
   - `Err(other)` -> return `Err(VerifyKeychainError::Unexpected(other.to_string()))`
   - Wrap execute body in `info_span!("usecase.verify_keychain_access.execute")` with `.instrument(span).await`
