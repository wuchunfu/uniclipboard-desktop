1. Create `src/components/setting/KeyRecorder.tsx`:
   - Import `useRecordHotkeys` from `react-hotkeys-hook`
   - Import `normalizeHotkey` from `@/shortcuts/normalize`
   - Import `getCandidateKeyIssues`, `resolveShortcuts` from `@/shortcuts/conflicts`
   - Import `SHORTCUT_DEFINITIONS` from `@/shortcuts/definitions`
   - Props: `shortcutId: string`, `scope: ShortcutScope`, `currentOverrides: ShortcutKeyOverrides`, `onConfirm: (key: string, clearedIds?: string[]) => void`, `onCancel: () => void`
   - On mount, call `start()` to begin recording
   - Convert recorded `keys` Set to hotkey string: `Array.from(keys).join('+')`, then `normalizeHotkey()` for display and comparison
   - PITFALL: `useRecordHotkeys` returns a Set of individual key tokens. Join with `+` then normalize.
   - Escape detection: if keys Set contains 'escape', call `stop()` then `onCancel()`. Must check this BEFORE running conflict detection.
   - Run `getCandidateKeyIssues(resolvedShortcuts, { id: shortcutId, scope, key: normalizedKey })` on each key change for real-time conflict feedback
   - Display: Show recorded key as kbd badges (reuse KeyBadge pattern from ShortcutRow), plus conflict info below if issues exist
   - For `level: 'error'` conflicts: Show conflict message and two buttons: "Apply anyway (clears conflict)" and "Cancel"
   - For `level: 'warning'` or `level: 'info'`: Show info but allow confirm without clearing
   - When user confirms with error-level conflict: pass `clearedIds` (the relatedIds from the error issue) to `onConfirm` so parent can clear those overrides
   - Confirm button calls `stop()` then `onConfirm(normalizedKey, clearedIds)`
   - Cancel button calls `stop()` then `onCancel()`
   - PITFALL: During recording, app shortcuts may fire. Use `e.preventDefault()` and `e.stopPropagation()` where needed. The `useRecordHotkeys` hook likely handles preventDefault internally.
   - Styling: border ring animation or pulsing border to indicate recording state (Claude's discretion on exact animation)

2. Update `src/components/setting/ShortcutRow.tsx`:
   - Add state: `isRecording: boolean` (local state)
   - When `isRecording` is true, render `KeyRecorder` inline instead of the key badge
   - `onEdit` sets `isRecording = true`
   - `onConfirm` from KeyRecorder: call parent's `onOverrideChange(id, newKey, clearedIds)` then set `isRecording = false`
   - `onCancel` from KeyRecorder: set `isRecording = false`
   - Pass `currentOverrides` through from ShortcutsSection
   - `onReset` calls parent's `onResetShortcut(id)` directly
   - Update Props to receive: `onOverrideChange: (id, key, clearedIds?) => void`, `onResetShortcut: (id) => void`, `currentOverrides: ShortcutKeyOverrides`

3. Update `src/components/setting/ShortcutsSection.tsx`:
   - Replace stub handlers with real persistence logic:
   - `handleOverrideChange(id: string, newKey: string, clearedIds?: string[])`:
     - Start from current overrides: `{ ...setting.keyboard_shortcuts }`
     - Set `overrides[id] = newKey`
     - If `clearedIds` provided, delete each from overrides AND if the cleared shortcut's default key equals the new key being set, just delete the override entry (effectively resetting it to default)
     - Call `updateKeyboardShortcuts(overrides)`
   - `handleResetShortcut(id: string)`:
     - Copy current overrides, delete `overrides[id]`, call `updateKeyboardShortcuts(overrides)`
   - `handleResetAll()`:
     - Call `updateKeyboardShortcuts({})` to clear all overrides
   - Pass handlers down to ShortcutRow components
   - "Reset All Shortcuts" button enabled when `Object.keys(setting?.keyboard_shortcuts ?? {}).length > 0`
