1. **Outbound chunked write with progress** -- Modify `execute_business_stream` to write the payload in 256KB chunks instead of a single `write_all`, emitting progress events via `event_tx`:

Replace the current pattern:

```rust
match timeout(write_timeout, stream.write_all(data)).await { ... }
```

With a chunked write loop that emits progress:

```rust
// Write payload in 256KB network chunks with progress tracking
const NETWORK_CHUNK_SIZE: usize = 256 * 1024;
let total = data.len() as u64;
let total_chunks = ((data.len() + NETWORK_CHUNK_SIZE - 1) / NETWORK_CHUNK_SIZE) as u32;
let mut written = 0u64;
let mut chunks_completed = 0u32;
// Throttle: emit progress at most every 100ms
let mut last_progress = std::time::Instant::now();
let write_result = timeout(write_timeout, async {
    for chunk in data.chunks(NETWORK_CHUNK_SIZE) {
        stream.write_all(chunk).await?;
        written += chunk.len() as u64;
        chunks_completed += 1;
        // Emit progress: throttle to ~10/sec, always emit first and last chunk
        if chunks_completed == 1 || chunks_completed == total_chunks || last_progress.elapsed() >= Duration::from_millis(100) {
            let _ = event_tx.send(NetworkEvent::TransferProgress(TransferProgress {
                transfer_id: transfer_id.clone(),
                peer_id: peer.to_string(),
                direction: TransferDirection::Sending,
                chunks_completed,
                total_chunks,
                bytes_transferred: written,
                total_bytes: total,
            })).await;
            last_progress = std::time::Instant::now();
        }
    }
    stream.flush().await?;
    Ok::<(), std::io::Error>(())
}).await;
```

The `event_tx` (mpsc::Sender<NetworkEvent>) is already available in the network adapter event loop. Pass it into `execute_business_stream` as a parameter (or access from `self`). The `transfer_id` can be extracted from the V3 header bytes (offset 5..21) or generated as a display string from the peer+timestamp.

This produces byte-identical wire output. The receiver's `read_to_end` is agnostic to write chunking since TCP/yamux handles reassembly.

2. **Inbound progress-reporting read** -- Modify the `spawn_blocking` inbound V3 path to read chunk-by-chunk using the V3 wire format structure instead of `read_to_end`:

   Replace the current `read_to_end` pattern in the inbound V3 clipboard handler:

   ```rust
   let encrypted = tokio::task::spawn_blocking(move || {
       use std::io::Read;
       use tokio_util::io::SyncIoBridge;
       let mut sync_reader = SyncIoBridge::new(reader);
       let mut buf = Vec::new();
       sync_reader.read_to_end(&mut buf)?;
       Ok::<Vec<u8>, anyhow::Error>(buf)
   })
   ```

   With a V3-header-aware chunked read that still accumulates the full encrypted payload (needed for decrypt) but reads chunk-by-chunk and emits progress events:

   ```rust
   // Clone event_tx for use in spawn_blocking (send progress via channel)
   let progress_event_tx = event_tx.clone();
   let peer_id_str = peer_id.to_string();
   let encrypted = tokio::task::spawn_blocking(move || {
       use std::io::Read;
       use tokio_util::io::SyncIoBridge;
       let mut sync_reader = SyncIoBridge::new(reader);

       // Read V3 header (37 bytes) to get total_chunks and transfer_id
       let mut header = [0u8; 37];
       sync_reader.read_exact(&mut header)?;

       let total_chunks = u32::from_le_bytes(header[25..29].try_into().unwrap());
       let transfer_id_bytes = &header[5..21];
       let transfer_id = hex::encode(transfer_id_bytes);

       // Accumulate: header + per-chunk (4-byte len prefix + ciphertext)
       let mut buf = Vec::from(&header[..]);
       let mut bytes_received = 37u64; // header bytes
       let mut last_progress = std::time::Instant::now();
       for chunk_idx in 0..total_chunks {
           let mut len_buf = [0u8; 4];
           sync_reader.read_exact(&mut len_buf)?;
           let ct_len = u32::from_le_bytes(len_buf) as usize;
           buf.extend_from_slice(&len_buf);
           let mut ct_buf = vec![0u8; ct_len];
           sync_reader.read_exact(&mut ct_buf)?;
           buf.extend_from_slice(&ct_buf);
           bytes_received += 4 + ct_len as u64;
           let chunks_completed = chunk_idx + 1;

           // Emit progress: throttle to ~10/sec, always emit first and last chunk
           if chunks_completed == 1 || chunks_completed == total_chunks || last_progress.elapsed() >= std::time::Duration::from_millis(100) {
               let _ = progress_event_tx.blocking_send(NetworkEvent::TransferProgress(TransferProgress {
                   transfer_id: transfer_id.clone(),
                   peer_id: peer_id_str.clone(),
                   direction: TransferDirection::Receiving,
                   chunks_completed,
                   total_chunks,
                   bytes_transferred: bytes_received,
                   total_bytes: 0, // unknown until fully read
               }));
               last_progress = std::time::Instant::now();
           }
       }
       Ok::<Vec<u8>, anyhow::Error>(buf)
   })
   ```

   This is structurally identical -- the accumulated `buf` contains the same bytes as `read_to_end` would produce. The `transfer_decryptor.decrypt(&encrypted, &master_key)` call afterwards works unchanged.

   **Important**: Maintain all existing error handling, timeout wrapping, and stream ownership patterns. The `SyncIoBridge` pattern is preserved. Only the read strategy inside `spawn_blocking` changes.
