**Create `src/hooks/useDeviceDiscovery.ts`:**

Define a `DiscoveredPeer` type: `{ id: string; deviceName: string | null; device_type: string }`. Note: `deviceName` is raw from the backend -- it may be `null` or a real name. The hook does NOT apply any fallback. The render layer is responsible for displaying a localized fallback when `deviceName` is null.

Define `ScanPhase = 'scanning' | 'hasDevices' | 'empty'`.

Define `UseDeviceDiscoveryOptions`:

```typescript
interface UseDeviceDiscoveryOptions {
  onError?: (error: Error) => void
}
```

Note: NO `fallbackDeviceName` option. The hook stores raw data only.

Export `useDeviceDiscovery(active: boolean, options?: UseDeviceDiscoveryOptions)` hook that returns `{ peers: DiscoveredPeer[], scanPhase: ScanPhase, resetScan: () => void }`.

**Reactivity contract for options (CRITICAL -- ref-based approach):**

The `options.onError` is stored in a `useRef` value and synced on every render. The main useEffect depends ONLY on `[active]` -- it never teardowns/re-subscribes when options change. Callbacks read from the ref, not from closure-captured options.

```typescript
const onErrorRef = useRef(options?.onError)

// Sync ref every render
onErrorRef.current = options?.onError
```

**The hook does NOT import `toast` from `sonner`.** All error presentation is delegated to the caller via `onError`.

Implementation details:

- State: `peers` (DiscoveredPeer[]), `scanPhase` (ScanPhase, initial 'scanning')
- Refs: `timeoutRef` for the 10-second timer, `onErrorRef` for error callback

**Deactivation/re-entry reset (CRITICAL):**
When `active` transitions from `true` to `false`, the hook MUST reset state to prevent stale data on re-entry:

- In the main useEffect cleanup: set `setPeers([])` and `setScanPhase('scanning')` BEFORE clearing listeners/timeout.
- This ensures that when `active` becomes `true` again, the UI starts fresh in scanning state with an empty peer list -- matching the current behavior at SetupPage.tsx line 210 (`setIsScanningInitial(true)`).

- `resetScan` callback: clear existing timeout, set peers to `[]`, set scanPhase to 'scanning', set new 10s timeout, call `getP2PPeers()` again for fresh load (with error handling -- see below).

**Error handling for getP2PPeers() (CRITICAL):**
Both initial load and `resetScan` call `getP2PPeers()`. Both MUST wrap in try/catch:

- On error: `console.error('Failed to fetch peers:', error)` and call `onErrorRef.current?.(error instanceof Error ? error : new Error(String(error)))`. Do NOT import toast/sonner -- the caller handles UI presentation.
- On error: set `scanPhase` to 'scanning' (do NOT transition to 'empty' on fetch failure -- the 10s timeout will handle that if no events arrive). Do NOT set scanPhase to 'hasDevices' on error.
- The hook remains functional after error -- event listeners still work and can populate the list.

This delegates i18n-aware error toasting to the caller (SetupPage), keeping the hook free of i18n and toast dependencies.

Main useEffect (depends on `[active]` ONLY):

- Guard: `if (!active) return`
- Set `cancelled = false`
- Reset state at start: `setPeers([])`, `setScanPhase('scanning')` -- ensures clean slate on re-entry.
- Initial load: call `getP2PPeers()` wrapped in try/catch (see error handling above). On success, map results to DiscoveredPeer format: `{ id: p.peerId, deviceName: p.deviceName ?? null, device_type: 'desktop' }`. Store raw deviceName -- do NOT apply any fallback. If list has items, set scanPhase to 'hasDevices'.
- Start 10s timeout: `setTimeout(() => setScanPhase(prev => prev === 'scanning' ? 'empty' : prev), 10_000)` -- use functional updater to avoid stale closure per RESEARCH pitfall #1.
- Set up three event listeners using the SAME pattern as useClipboardEventStream (cancelled guard + async unlisten cleanup):
  1. `onP2PPeerDiscoveryChanged`:
     - If `discovered === true`: add/update peer in list (upsert by peerId), set scanPhase to 'hasDevices'
     - If `discovered === false`: remove peer by peerId. If list becomes empty after removal, set scanPhase to 'empty'
     - For the deviceName field: store raw `event.deviceName ?? null` (no fallback)

  2. `onP2PPeerNameUpdated`:
     - Update peer deviceName silently (find by peerId, update deviceName to `event.deviceName`). No scanPhase change per user decision.

  3. `onP2PPeerConnectionChanged`:
     - Do NOT remove devices on `connected: false` per user decision and RESEARCH recommendation. This is a silent update only -- no state change needed for discovery list.

- Cleanup: reset state (`setPeers([])`, `setScanPhase('scanning')`), set cancelled=true, clear timeout, call `.then(fn => fn())` on all three listener promises.

Export `ScanPhase` type (import from hook or redefine -- prefer importing from hook).

Update `JoinPickDeviceStepProps`:

- Add `peers: DiscoveredPeer[]` (import DiscoveredPeer from hook)
- Add `scanPhase: ScanPhase`

Keep extending `StepProps` for error/loading/direction.
