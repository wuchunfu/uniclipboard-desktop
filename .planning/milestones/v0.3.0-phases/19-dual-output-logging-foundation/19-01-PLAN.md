  <behavior>
    - init_tracing_subscriber(logs_dir, profile) creates a registry with console + JSON layers
    - Console layer uses pretty format with ChronoUtc timer, ansi colors, file/line/target info
    - JSON layer uses FlatJsonFormat with daily rolling file writer to logs_dir
    - Each layer has its own EnvFilter from the profile (per-layer filtering, not global)
    - When RUST_LOG is set, both layers use RUST_LOG-derived filter instead of profile filters
    - WorkerGuard for JSON file writer is stored in static OnceLock to prevent early drop
    - Daily rolling appender creates files named uniclipboard.json.YYYY-MM-DD
    - Function returns Ok(()) on success, Err if subscriber already registered
  </behavior>
  <action>
1. Create `src-tauri/crates/uc-observability/src/init.rs`:
   - Static guard storage: `static JSON_GUARD: OnceLock<WorkerGuard> = OnceLock::new();`
   - `pub fn init_tracing_subscriber(logs_dir: &Path, profile: LogProfile) -> anyhow::Result<()>`:
     a. Determine filters: if `RUST_LOG` is set, use `EnvFilter::try_from_default_env()?` for both layers. Otherwise use `profile.console_filter()` and `profile.json_filter()`.
     b. Console layer: `fmt::layer()` with `.with_timer(ChronoUtc)`, `.with_level(true)`, `.with_file(true)`, `.with_line_number(true)`, `.with_target(true)`, `.with_ansi(cfg!(not(test)))`, `.with_writer(std::io::stdout)`, `.with_filter(console_filter)`.
     c. JSON layer: create daily rolling appender via `tracing_appender::rolling::daily(logs_dir, "uniclipboard.json")`. Wrap with `tracing_appender::non_blocking()`. Store guard in `JSON_GUARD` OnceLock. Create `fmt::layer().event_format(FlatJsonFormat::new()).fmt_fields(JsonFields::new()).with_writer(non_blocking).with_filter(json_filter)`. Note: Use `JsonFields` as the field formatter so span fields are stored as parseable JSON in extensions — this is critical for FlatJsonFormat to extract structured span data.
     d. Compose: `registry().with(console_layer).with(json_layer).try_init()?;`
     e. Log the active profile: `tracing::info!(profile = %profile, "Tracing initialized with dual output");`
     f. Return Ok(()).
   - Note: Sentry layer is NOT included here. Sentry initialization remains in the caller (main.rs / uc-tauri) since it depends on app-level config (SENTRY_DSN). The caller can add a sentry layer to the registry separately, or — since the subscriber is already initialized — sentry integration will be handled via the existing sentry-tracing approach in main.rs. Document this in the function's doc comment.
