## Decisions Made

- Live daemon child ownership is now a dedicated managed Tauri state instead of being mixed into `DaemonBootstrapOwnershipState`.
- `ProbeOutcome::Compatible` explicitly clears GUI-owned daemon state so independently started compatible daemons are never marked for later cleanup.
- `WindowEvent::CloseRequested` remains untouched; daemon lifecycle ownership is wired independently from hide-to-tray behavior.
