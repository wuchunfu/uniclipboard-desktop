## Current Problem

In uc-tauri/src/commands/pairing.rs, the `get_p2p_peers` command aggregates multiple use cases:

```rust
let discovered = runtime.usecases().list_discovered_peers().execute().await?;
let connected = runtime.usecases().list_connected_peers().execute().await?;
let connected_map = connected_peer_ids(&connected);
// Merge discovered with connected status
```

This cross-use-case aggregation happens in the command layer, making it:

1. Not reusable from CLI
2. Duplicated if another command needs similar logic
3. Not testable as a unit

## Target Solution

Add a new `GetP2pPeersSnapshot` use case in uc-app that:

1. Calls list_discovered_peers()
2. Calls list_connected_peers()
3. Merges them into a unified snapshot

Then update both Tauri commands and CLI to use this shared use case.

## Key Interfaces

From uc-app/src/usecases/mod.rs:

- `CoreUseCases::list_discovered_peers() -> ListDiscoveredPeers`
- `CoreUseCases::list_connected_peers() -> ListConnectedPeers`
- `CoreUseCases::list_paired_devices() -> ListPairedDevices`

New use case to create:

- `GetP2pPeersSnapshot` - combines discovered + connected peers
