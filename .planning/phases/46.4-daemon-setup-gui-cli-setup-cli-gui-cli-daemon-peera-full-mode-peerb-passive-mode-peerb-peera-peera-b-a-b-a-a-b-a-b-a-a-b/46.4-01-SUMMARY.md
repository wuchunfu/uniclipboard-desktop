**Daemon-owned setup HTTP routes with camelCase state metadata, fixed next-step hints, and deterministic regression coverage for CLI-facing setup transport**

## Accomplishments

- Added authenticated daemon `/setup/state`, `/setup/host`, `/setup/join`, `/setup/select-peer`, `/setup/confirm-peer`, `/setup/submit-passphrase`, and `/setup/cancel` routes that delegate to the existing `SetupOrchestrator`.

## Decisions Made

- Injected `SetupOrchestrator` into `DaemonApiState` instead of rehydrating from `CoreRuntime` in handlers, keeping setup truth explicitly daemon-owned at the transport boundary.
- Used the existing serialized `SetupState` as the `state` payload and layered transport metadata beside it, so CLI can reuse the same semantic state model as GUI.
- Used a fake pairing facade only inside transport tests to keep HTTP regressions deterministic while leaving production code wired to the real daemon/runtime setup path.
