**Typed uc-core realtime topics, payload-specific events, and a single daemon://realtime frontend envelope for downstream bridge and consumer work**

## Accomplishments

- Added `uc-core::ports::realtime` with `RealtimeTopicPort`, stable topics, typed inbound events, and typed frontend payload enums/structs.
- Extended `HostEvent` with `Realtime(RealtimeFrontendEvent)` so downstream layers can emit the unified frontend contract without tauri dependencies.
- Added isolated `uc-core` contract tests for topic stability, typed payload coverage, and the `daemon://realtime` envelope contract.

## Decisions Made

- Kept the frontend envelope serializer-free in `uc-core`; downstream tauri code will own serialization while app/core code depend only on typed Rust models.
- Used a raw identifier field (`r#type`) in `RealtimeFrontendEvent` so the Rust model preserves the frontend wire key name `type` exactly.
