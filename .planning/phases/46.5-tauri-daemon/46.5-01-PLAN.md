<objective>
把“谁拥有本地pairing runtime”从隐式事实改成composition-root级别的显式配置，并在GUI builder上切断真实本地pairing transport。

Purpose: 让GUI不再通过“忽略本地事件”假装daemon-owned，而是在`uc-bootstrap` + `uc-platform`层真正不创建本地pairing runtime。
Output: pairing runtime owner配置、GUI disabled pairing transport装配、以及GUI/daemon owner差异化测试。
</objective>

    - Test 1: 传入 `PairingRuntimeOwner::ExternalDaemon` 时，`spawn_swarm()` 不创建 `PairingStreamService`，不会启动 pairing accept loop，且不会注册 `/uniclipboard/pairing-stream/1.0.0` handler/protocol。
    - Test 2: 传入 `PairingRuntimeOwner::CurrentProcess` 时，现有本地pairing行为保持可用，`pairing_service` 仍会被初始化，并继续注册 `/uniclipboard/pairing-stream/1.0.0`。
    - Test 3: `DisabledPairingTransport` 的 `open_pairing_session`、`send_pairing_on_session`、`close_pairing_session`、`unpair_device` 全部返回固定错误字符串 `local pairing runtime is disabled in this process`。

  <action>
    1. 在 `src-tauri/crates/uc-platform/src/adapters/network.rs` 中新增 `PairingRuntimeOwner` 枚举，固定包含两个值：
       - `CurrentProcess`
       - `ExternalDaemon`
    2. 同文件新增 `DisabledPairingTransport`，实现 `PairingTransportPort`，四个方法全部返回 `anyhow!("local pairing runtime is disabled in this process")`；不要静默 `Ok(())`。
    3. 在 `src-tauri/crates/uc-platform/src/adapters/libp2p_network.rs` 中给 `Libp2pNetworkAdapter` 增加 `pairing_runtime_owner: PairingRuntimeOwner` 字段，并把 `Libp2pNetworkAdapter::new(...)` 的最后一个参数改成这个 owner。
    4. 在 `spawn_swarm()` 中仅当 owner 为 `PairingRuntimeOwner::CurrentProcess` 时才执行下面四步：
       - 创建 `PairingStreamService::new(...)`
       - 调用 `pairing_service.spawn_accept_loop()`
       - 注册 `/uniclipboard/pairing-stream/1.0.0` 对应的 pairing stream handler/protocol
       - 把 `self.pairing_service` 写成 `Some(pairing_service)`
       当 owner 为 `ExternalDaemon` 时，必须跳过这四步，并保持 `self.pairing_service == None`。
    5. 保留文件传输、剪贴板、peer discovery 的既有启动逻辑，不要因为禁用pairing runtime而影响 `FileTransferService` 或 `stream_control` 初始化。
    6. 在 `src-tauri/crates/uc-platform/src/adapters/mod.rs` 里导出 `PairingRuntimeOwner` 和 `DisabledPairingTransport`，供 `uc-bootstrap` 装配使用。
    7. 在 `libp2p_network.rs` 的现有测试区补两组明确命名的测试：
       - `pairing_runtime_disabled_does_not_initialize_pairing_service`
       - `pairing_runtime_disabled_does_not_register_pairing_protocol`
       - `pairing_runtime_current_process_initializes_pairing_service`
       并在 `network.rs` 补 `disabled_pairing_transport_returns_fixed_error`。
  </action>
