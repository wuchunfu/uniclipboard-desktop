**9. Create `src-tauri/crates/uc-daemon/src/state.rs`:**

RuntimeState holds **only pure snapshot data** (start time + cached worker statuses). No `DaemonWorker` references — `DaemonApp` owns workers and periodically updates the snapshot. This makes `RuntimeState` fully `Send + Sync` without trait object concerns, and RPC reads never contend with worker lifecycle.

```rust
use std::time::Instant;
use crate::rpc::types::WorkerStatus;

pub struct RuntimeState {
    start_time: Instant,
    worker_statuses: Vec<WorkerStatus>,
}

impl RuntimeState {
    pub fn new(initial_statuses: Vec<WorkerStatus>) -> Self {
        Self {
            start_time: Instant::now(),
            worker_statuses: initial_statuses,
        }
    }
    pub fn uptime_seconds(&self) -> u64 {
        self.start_time.elapsed().as_secs()
    }
    pub fn worker_statuses(&self) -> &[WorkerStatus] {
        &self.worker_statuses
    }
    pub fn update_worker_statuses(&mut self, statuses: Vec<WorkerStatus>) {
        self.worker_statuses = statuses;
    }
}
```

`DaemonApp` is responsible for calling `state.write().await.update_worker_statuses(...)` after starting/stopping workers, and can periodically refresh health via `worker.health_check()` on its own `Vec<Arc<dyn DaemonWorker>>`.
