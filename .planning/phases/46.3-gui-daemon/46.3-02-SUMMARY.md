# Phase 46.3 Plan 02: Bounded Replacement Summary

**Profile-aware daemon PID metadata plus a one-shot incompatible-daemon replacement path, so GUI startup can replace stale local daemons without becoming a generic supervisor**

## Accomplishments

- Added `process_metadata.rs` with profile-aware PID file helpers and bound PID metadata lifecycle to daemon startup/shutdown through a dedicated guard in `app.rs`.
- Added `DaemonBootstrapOwnershipState` and upgraded bootstrap orchestration to perform at most one incompatible-daemon terminate/wait/spawn cycle before failing deterministically.
- Added bootstrap contract tests for replace-once, bounded failure, and absent-direct-spawn behavior without relying on real host process enumeration.

## Files Created/Modified

- `src-tauri/crates/uc-daemon/src/process_metadata.rs` - profile-aware PID path, write/remove/read helpers, and daemon PID tests.
- `src-tauri/crates/uc-daemon/src/app.rs` - daemon PID guard that writes current PID after bootstrap and removes PID metadata on shutdown or early return.
- `src-tauri/crates/uc-daemon/src/lib.rs` - exports `process_metadata` for GUI bootstrap reuse.
- `src-tauri/crates/uc-tauri/src/bootstrap/runtime.rs` - dedicated ownership state for replacement attempts, spawned child PID, and last incompatible reason.
- `src-tauri/crates/uc-tauri/src/bootstrap/run.rs` - bounded replacement flow, terminate-by-pid-file logic, and hookable bootstrap helper.
- `src-tauri/crates/uc-tauri/tests/daemon_bootstrap_contract.rs` - deterministic regression coverage for replace-once and fail-bounded semantics.

## Decisions Made

- Kept replacement scope narrow by reading only UniClipboard’s own PID metadata, never by searching arbitrary host processes.
- Recorded replacement ownership separately from connection state so later startup gating can consume retry/replacement facts without polluting connection info.
- Used OS-native `kill -TERM` / `taskkill` command paths only behind the PID-file gate, preserving the “expected local endpoint only” boundary.
