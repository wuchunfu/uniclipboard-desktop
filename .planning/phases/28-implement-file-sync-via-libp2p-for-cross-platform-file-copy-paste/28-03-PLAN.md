# Plan 28-03: Protocol registration, port trait, network events, database schema, and wiring

## Objective

Register the file transfer protocol ID, define the FileTransportPort trait, extend network and platform events, create the database schema for file transfers, and wire the port into the dependency graph.

## must_haves

- [ ] `ProtocolId::FileTransfer` variant with `/uniclipboard/file-transfer/1.0.0`
- [ ] `FileTransportPort` trait with send/receive methods
- [ ] File transfer event variants in `NetworkEvent`
- [ ] `PlatformEvent::FileCopied` variant
- [ ] Database migration creating `file_transfer` table
- [ ] `file_transfer` field in `NetworkPorts` struct
- [ ] All crates compile clean (`cargo check`)

## Tasks

<task id="28-03-01">
<title>Add protocol ID, FileTransportPort trait, and event extensions</title>
<description>
1. **ProtocolId** (`uc-core/src/network/protocol_ids.rs`):
   - Add `FileTransfer` variant to `ProtocolId` enum
   - Map to `"/uniclipboard/file-transfer/1.0.0"` in `as_str()`
   - Update test to verify the new variant string

2. **FileTransportPort** (`uc-core/src/ports/file_transport.rs` — create new file):
   - Define trait:
     ```rust
     #[async_trait]
     pub trait FileTransportPort: Send + Sync {
         async fn send_file_announce(&self, peer_id: &str, announce: FileTransferMessage) -> Result<()>;
         async fn send_file_data(&self, peer_id: &str, data: FileTransferMessage) -> Result<()>;
         async fn send_file_complete(&self, peer_id: &str, complete: FileTransferMessage) -> Result<()>;
         async fn cancel_transfer(&self, peer_id: &str, cancel: FileTransferMessage) -> Result<()>;
     }
     ```
   - Add `pub mod file_transport;` to `uc-core/src/ports/mod.rs`
   - Add `pub use file_transport::FileTransportPort;` to re-exports

3. **NetworkEvent extensions** (`uc-core/src/network/events.rs`):
   - Add variants:
     ```rust
     FileTransferStarted { transfer_id: String, peer_id: String, filename: String, file_size: u64 },
     FileTransferCompleted { transfer_id: String, peer_id: String, filename: String },
     FileTransferFailed { transfer_id: String, peer_id: String, error: String },
     FileTransferCancelled { transfer_id: String, peer_id: String, reason: String },
     ```

4. **PlatformEvent extension** (`uc-platform/src/ipc/event.rs`):
   - Add `FileCopied { file_paths: Vec<String> }` variant

5. **Tests**:
   - Verify `ProtocolId::FileTransfer.as_str()` returns correct string
   - Verify `NetworkEvent` file transfer variants serialize/deserialize correctly
</description>

<task id="28-03-02">
<title>Create database migration and wire FileTransportPort into deps</title>
<description>
1. **Database migration** — Create directory `src-tauri/crates/uc-infra/migrations/2026-03-13-000001_create_file_transfer/`:

   `up.sql`:
   ```sql
   CREATE TABLE file_transfer (
       transfer_id TEXT PRIMARY KEY NOT NULL,
       filename TEXT NOT NULL,
       file_size BIGINT NOT NULL,
       content_hash TEXT NOT NULL,
       status TEXT NOT NULL DEFAULT 'pending',
       source_device TEXT NOT NULL,
       batch_id TEXT,
       cached_path TEXT,
       created_at_ms BIGINT NOT NULL,
       updated_at_ms BIGINT NOT NULL
   );

   CREATE INDEX idx_file_transfer_status ON file_transfer(status);
   CREATE INDEX idx_file_transfer_batch ON file_transfer(batch_id);
   CREATE INDEX idx_file_transfer_created ON file_transfer(created_at_ms);
   ```

   `down.sql`:
   ```sql
   DROP TABLE IF EXISTS file_transfer;
   ```
