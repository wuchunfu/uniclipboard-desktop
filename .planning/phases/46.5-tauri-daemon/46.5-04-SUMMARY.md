# Phase 46.5 Plan 04: Command Shell Cutover Summary

**Daemon-backed Tauri pairing/setup command shell with legacy backend `p2p-command-error` emission removed**

## Performance

- **Duration:** 13 min
- **Started:** 2026-03-22T04:37:26Z
- **Completed:** 2026-03-22T04:50:55Z
- **Tasks:** 2
- **Files modified:** 4

## Accomplishments

- `commands/pairing.rs` now reads paired devices and unpairs only through daemon query/pairing clients, while keeping local-only device info and sync settings paths untouched.
- `commands/setup.rs` now keeps the existing Tauri command names but forwards all setup actions to daemon `/setup/*` routes and deserializes `state` payloads back into `SetupState`.
- `daemon_command_shell.rs` locks the pairing/setup shell routes and the local passphrase mismatch contract, while asserting backend `p2p-command-error` emission stays removed.

## Files Created/Modified

- `src-tauri/crates/uc-core/src/setup/state.rs` - allows command shell code to deserialize daemon `state` payloads into `SetupState`
- `src-tauri/crates/uc-tauri/src/commands/pairing.rs` - removes local paired-device/unpair active paths and legacy backend `p2p-command-error` emission
- `src-tauri/crates/uc-tauri/src/commands/setup.rs` - converts setup commands into daemon-backed wrappers while preserving the mismatch fallback contract
- `src-tauri/crates/uc-tauri/tests/daemon_command_shell.rs` - adds scripted daemon-shell regressions for pairing and setup commands

## Decisions Made

- Kept the frontend invoke surface stable by preserving the existing Tauri command names and returning `SetupState`/`PairedPeer` shapes from daemon responses.
- Mapped daemon `PairedDeviceDto` to `PairedPeer` with intentionally empty `shared_secret`, `paired_at`, and `last_known_addresses` fields to preserve the current frontend read contract without reintroducing local truth.
- Preserved the local passphrase mismatch fast-fail path in `submit_passphrase` so validation still happens before any daemon transport dependency.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Added `Deserialize` support to `SetupState`**

- **Found during:** Task 1 (daemon setup command shell cutover)

- **Fix:** Derived `serde::Deserialize` for `uc_core::setup::SetupState`.
- **Files modified:** `src-tauri/crates/uc-core/src/setup/state.rs`
- **Verification:** `cargo test -p uc-tauri --test daemon_command_shell -- --test-threads=1`
- **Committed in:** `aec5da8f`

## Issues Encountered

- The new IPC-based regression test initially deadlocked because `tauri::test::get_ipc_response` is synchronous while the fake daemon server ran on Tokio. The test was switched to `#[tokio::test(flavor = "multi_thread")]`, which resolved the scheduling stall without changing production code.
