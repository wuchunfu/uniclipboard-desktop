List them all before proceeding.

    2. Create `action_executor.rs` with a `SetupActionExecutor` struct:
       - Move port `Arc` fields that are ONLY used by action methods from `SetupOrchestrator` into `SetupActionExecutor`
       - The executor should own: `initialize_encryption`, `mark_setup_complete`, `app_lifecycle`, `discovery_port`, `network_control`, `crypto_factory`, `pairing_transport`, `transport_port`, `proof_port`, `timer_port`, `persistence_port`, `setup_event_port`, and any other ports only used by action execution
       - The orchestrator should retain: `pairing_orchestrator`, `space_access_orchestrator`, `setup_status`, `context` (SetupContext), and session/state management fields
       - Move each action-handler method body into `SetupActionExecutor` methods with the same name and signature
       - The executor methods may need references to orchestrator-owned state (e.g., context). Pass these as method parameters, NOT by storing a back-reference to the orchestrator (avoids circular references)

    3. Update `SetupOrchestrator`:
       - Add `action_executor: Arc<SetupActionExecutor>` field
       - Update `new()` constructor to create `SetupActionExecutor` with the moved port fields. The constructor parameter count should DECREASE (net reduction), not increase. The executor absorbs parameters.
       - Replace each inline action-handler call with `self.action_executor.method_name(...)` delegation
       - Keep the public API methods (new_space, join_space, etc.) unchanged in signature

    4. Update `mod.rs` to add `mod action_executor;` (no need to pub-export — it's an internal implementation detail)

    CRITICAL pitfalls (from research):
    - Share the same `Arc` instances between orchestrator and executor. Do NOT clone-and-create-new.
    - If orchestrator holds `Arc<Mutex<Option<T>>>` for session state, the executor must borrow it via method params, NOT own a separate copy.
    - Extract shared types (errors, events) if needed to avoid circular imports. The `SetupError` enum should stay in `orchestrator.rs` or move to a `types.rs` submodule if the executor needs it. Do NOT duplicate it.
